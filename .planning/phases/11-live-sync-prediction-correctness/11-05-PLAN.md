<objective>
Wire the cross-night rotation window and its reset control into the app tier (the pure `currentRunShowSets` + `runGapDays` come from plan 11-03):
- **PRED-01:** replace the hardcoded `[]` 3rd argument to `buildShowContext` (`useShowSession.ts:123`) with the current run's prior-show song sets — a `useLiveQuery` reads finalized `trackedShows` + their entries, and `currentRunShowSets(...)` groups them by `config.runGapDays`. This finally feeds the already-correct `rotationSuppression`, so a song played earlier in the run sinks in tonight's ranked predictions (implicit rank drop, D-11 — no new prediction UI).

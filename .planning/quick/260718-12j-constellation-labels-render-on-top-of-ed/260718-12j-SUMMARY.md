# Quick 260718-12j: Constellation labels render on top of edges/orbs Summary

Moved the constellation song-name label drawing out of `nodeCanvasObject` into a
dedicated `drawNodeLabel` helper invoked from a new `onRenderFramePost` top pass, so
every visible label paints after all links, arrows, and orbs — never overpainted by a
later node's fill and never buried under edge lines.

## What Changed

- **Removed** the "5. Label" block from `nodeCanvasObject` (the `forced`/`showLabel`
  computation, font/color/text setup, and the `ctx.fillText(...)`). The node pass now
  keeps only blocks 1-4 (fill, sighting ring, focus ring, count pill) plus its own
  `ctx.save()`/`ctx.restore()`.
- **Added** an inner `drawNodeLabel(node, ctx, globalScale)` helper closing over
  `focusId`, `neighborIds`, `isInNeighborhood`, `isNodeVisible`, `topKIds`, `overlay`,
  `sightingsFor`, and `config`. It replicates the exact per-node state the node pass
  computed:
  - early-return on null `x`/`y`;
  - **re-added `!isNodeVisible(node.id)` guard** — required because `onRenderFramePost`
    iterates all nodes manually and does NOT get the `nodeVisibility` filter;
  - depth-scaled radius `r` via the same `radiusFor(...) * lerp(DEPTH_RADIUS_FAR,
    DEPTH_RADIUS_NEAR, z)` formula (label y-offset);
  - combined alpha identical to the node pass: `max(DEPTH_ALPHA_FLOOR,
    min(focusAlpha, dexAlpha) * depthAlpha)`, wrapped in `ctx.save()`/`ctx.restore()`;
  - identical `showLabel` gate (`forced || globalScale >= LABEL_ZOOM_THRESHOLD ||
    topKIds.has(id)`);
  - byte-identical font/color/ellipsize/`fillText`.
- **Wired** `onRenderFramePost={(ctx, globalScale) => { for (const raw of
  graphData.nodes) drawNodeLabel(raw as FgNode, ctx, globalScale); }}` on
  `<ForceGraph2D>`, placed next to the node draw callbacks.

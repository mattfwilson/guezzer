# Phase 18 — UI Design Contract

> Accounts & Offline-Safe Identity. Visual and interaction contract for the auth/identity seam —
> the name-picker sign-in, the offline-safe boot gate, the header identity avatar + sign-out sheet,
> the "reconnecting…" affordance, and the "Gizz With Friends" rebrand chrome. Generated by
> gsd-ui-researcher; verified by gsd-ui-checker.
>
> **Source discipline:** every token below is either (a) DETECTED from the shipped design system
> (`packages/app/src/styles.css` `@theme`, `config.ts`, `SyncDot.tsx`) or (b) DERIVED from the 18
> locked decisions in `18-CONTEXT.md` (D-01..D-18) and `18-RESEARCH.md`. **No new visual language is
> introduced** — this phase reuses the dark galaxy aesthetic, the 60/30/10 split, the dark-on-light
> orb glyph idiom, and the existing `SyncDot` connection vocabulary. The ONLY net-new visual
> primitive is the deterministic identity color palette (D-13), which is deliberately a fresh,
> decoupled family. Nothing was re-asked of the user; all contract fields were pre-populated from
> upstream artifacts.

## Spacing Scale

Tailwind v4 default 4px scale (detected in use across `AppShell`, `GamesView`, `SettingsView`).
All multiples of 4.

| Token | Value | Usage (this phase) |
|-------|-------|--------------------|
| xs | 4px | Avatar initials inset, inline glyph → text gap |
| sm | 8px | Password-field → error-line gap, roster-row icon → name gap, sheet row internal gap |
| md | 16px | Sign-in screen horizontal padding (`px-4`), identity-sheet padding, between name-picker and password field |
| lg | 24px | Vertical rhythm between sign-in blocks (wordmark → roster → password), sheet section breaks |
| xl | 32px | Sign-in screen top padding, gap between roster grid and submit CTA |
| 2xl | 48px | Sign-in wordmark → roster vertical breathing room on tall viewports |
| 3xl | 64px | Scroll bottom padding on the sign-in screen above the safe-area gutter |

Exceptions:
- **44px minimum tap target** (`min-h-11` / `min-w-11`) on every interactive element — each roster
  name tap target, the password field, the submit CTA, the header identity avatar, the sign-out
  control in the sheet. Venue reality: fat-thumb, one-thumb, in the dark, possibly drunk-thumb
  (D-01). Matches the app-wide `min-h-11` precedent.
- **Header identity avatar diameter: 32px** (`h-8 w-8`) glyph inside a **44px tap target** — the
  glyph is small chrome (sits beside the existing menu button, `AppShell.tsx:46-53`), but its tap
  zone must still clear 44px (negative-margin / padding to reach the floor, mirroring the SyncDot
  tap-target idiom).
- **Roster name tap targets are LARGE** (D-01 — "large tap targets"): minimum **56px tall** rows
  (`min-h-14`), full-width, generously spaced (≥8px gap) so a friend can pick their name one-thumbed
  in the dark without mis-taps.
- **Safe-area insets** honored via `env(safe-area-inset-*)`; top-anchored surfaces (the sign-in
  screen, the identity sheet header if top-anchored) use `calc(env(safe-area-inset-top) + 12px)` —
  NEVER a bare inset (the doubled-inset bug the app already fixed).

## Typography

Detected sizes/weights from shipped components (`AppShell`, `SettingsView`, `DexHeader`). Exactly
**4 roles, 2 weights**. Weights: regular **400** + semibold **600** (no other weight in the app).

| Role | Size | Weight | Line Height | Usage (this phase) |
|------|------|--------|-------------|--------------------|
| Label | 14px (`text-[14px]`) | 400 / 600 | 1.25 (`leading-tight`) | Inline password error, "Forgot? Ask Matt to reset it" line, avatar initials (1–2 chars), field caption/placeholder |
| Body | 16px (`text-base`) | 400 | 1.5 (`leading-normal`) | "Connect once" screen body copy, roster names, identity-sheet display_name subline, sign-out label |
| Heading | 20px (`text-[20px]`) | 600 | 1.2 (`leading-tight`) | Sign-in screen heading ("Who's here?"), identity-sheet display_name (primary), "connect once" heading |
| Display | 28px (`text-[28px]`) | 600 | 1.2 | Sign-in "Gizz With Friends" wordmark on the sign-in screen (single element) |

Notes:
- **No `text-input` zoom trap:** the password `<input>` must render at **≥16px** (`text-base`) so
  iOS Safari does not auto-zoom on focus at a venue — a known one-thumb-entry footgun. Non-negotiable.
- Password field uses `type="password"` `autoComplete="current-password"` and, once a roster name is
  tapped, `autoComplete` username context is prefilled from the roster handle (D-01) so the OS
  keychain can offer to save it (password typed once per device).

## Color

Detected `@theme` tokens (`styles.css`). The 60/30/10 split is already the app's contract; this
phase does **not** introduce a new surface or accent hue. The identity palette (below) is a distinct
**data-identity** language (like rarity/tuning colors), NOT part of the 10% accent budget.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0c0c10` (`--color-surface`) | App/screen background; the sign-in screen background (may reuse the ambient blurred-cover / galaxy backdrop already shipped, offline-safe, reduced-motion-aware) |
| Secondary (30%) | `#17171f` (`--color-elevated`) | The identity bottom-sheet, the password field, roster name cards/rows, the inline-error container |
| Accent (10%) | `#f2c14e` (`--color-accent`) | **Reserved — see list below** |
| Destructive | `#ef4444` (`--color-destructive`) | **Not used this phase** — sign-out is NOT data-destructive (prior user's rows are retained on-device per D-09), so it uses neutral chrome, never the destructive red. No red confirm dialog. |

**Accent (`#f2c14e`) reserved for — explicit list (never "all interactive elements"):**
1. Focus rings (app-wide a11y, unchanged) — including on roster name buttons, the password field,
   the submit CTA, and the header avatar.
2. The **sign-in submit CTA** ("Sign in") — the single primary action on the sign-in screen may wear
   an accent fill (matches the shipped owner-approved accent-fill primary-CTA precedent, e.g. the
   pre-opener center orb `260717-cta`). Exactly one accent-filled element per screen.
3. The sign-in screen "Gizz With Friends" wordmark emphasis (optional, single element).

**NOT accent (identity / connection-status language — "data semantics never chrome", the app's B3
principle). These are NOT part of the 10% accent budget:**

| Semantic | Hex | Source precedent | Usage this phase |
|----------|-----|------------------|------------------|
| Identity palette (NEW, D-13) | `config.auth.IDENTITY_COLORS` — recommended fresh 6-hue **pastel/light** family (below) | mirrors `config.map.MEMBER_COLORS` idiom but **decoupled** (D-13) | The deterministic identity avatar fill (color+initials); reused by Phase 19 friend rows + Phase 20 presence dots |
| Online / connected green | `#22C55E` | `SyncDot` online (`ONLINE_GREEN`) | Reconnected/connected state of the identity connection dot (reuses SyncDot green) |
| Soft-warning / reconnecting amber | `#F59E0B` | `SyncDot` schema-drift (`DRIFT_AMBER`, "info, not error") | The AUTH-08 "reconnecting…" state (D-07) — a calm amber, never red, never a logout |
| Offline ring / muted | `#a1a1aa` (`--color-text-muted`) | `SyncDot` offline ring | The hollow offline ring; the "connect once" body copy; the "Ask Matt to reset it" subline |
| On-avatar initials text | `#0C0C10` (`ORB_TEXT_COLOR` / `RARITY_ORB_TEXT_COLOR`) | `show/tuningColor.ts:25`, `dex/rarityStyle.ts:48` | Dark-on-light initials on every identity avatar (D-12) |

**Recommended `config.auth.IDENTITY_COLORS` palette (Claude's-discretion per D-13, this is the
prescribed default — 6 hues, one spare for the 5-friend group):**

| # | Hex | Note |
|---|-----|------|
| 1 | `#7DD3FC` | light sky |
| 2 | `#FDA4AF` | light rose |
| 3 | `#86EFAC` | light green |
| 4 | `#FCD34D` | light amber |
| 5 | `#C4B5FD` | light violet |
| 6 | `#5EEAD4` | light teal |

**Palette rationale (decoupling + legibility):** the identity family is deliberately a **light/pastel
(~Tailwind-300) family**, distinct from the saturated (~400/500) data-semantic hues already in use
(caught-green `#22C55E`, accent-gold `#f2c14e`, amber `#F59E0B`, rarity tiers, tuning colors, and
the GizzMap `MEMBER_COLORS`). This gives identity avatars their own visual register AND guarantees
every hue clears **≥4.5:1 contrast with the `#0C0C10` initials** (WCAG 1.4.3) — verifiable and
required. The exact hues are the planner/executor's to tune within this family, but they MUST: be
config-level (`config.auth.IDENTITY_COLORS`), be selected by the pure `identityColorIndex(userId,
len)` hash (deterministic from the Supabase user id, never from display_name, never stored), and
clear 4.5:1 with `#0C0C10`.

Contrast: the identity initials use `#0C0C10` on a light palette (≥4.5:1, verify per hue). The
connection dot amber `#F59E0B` and green `#22C55E` both clear WCAG 1.4.11 non-text contrast on
`#17171F`/`#0C0C10` (already proven in `SyncDot`). No status is conveyed by color alone: the
"reconnecting…" state carries an `aria-label`/text affordance (extends the existing SyncDot
`aria-label` discipline), and the avatar always shows initials (identity never relies on hue alone).

## Interaction & Motion Contract

The engineering weight of this phase is interaction (the boot gate) more than pixels. The planner +
executor consume the specifics below. All motion reuses the established `@keyframes`-inside-
`@media (prefers-reduced-motion: no-preference)` idiom in `styles.css` and the shipped
`useReducedMotion()` hook.

### The offline-safe boot gate (AUTH-02 / D-05 / D-06 — the crux, highest-risk)
- **Zero-await first paint.** The gate interposes between `main.tsx` and `<App/>` (Research Pattern
  2, `App.tsx` ~76-95). It reads the **app-owned identity record** *synchronously* (localStorage /
  cached Dexie meta). There is **NO splash, NO spinner, NO `await getSession()` before first paint.**
  A restored identity paints the full app instantly, exactly as v1 boots today — this must not
  regress the shipped offline boot.
- **Gate on identity PRESENCE, not token validity (D-06).** An expired-but-present identity opens
  the full offline dex. `getSession()` / `onAuthStateChange` run only in the background to reconcile
  connectivity; they NEVER block boot and NEVER force the sign-in screen except on an **explicit**
  `SIGNED_OUT`.
- **No jarring transitions.** Sign-in → app lands on the app's current default hash route (D-17), no
  relocation. Sign-out → sign-in screen instantly, view torn down (D-10), no cross-fade that could
  flash the prior dex.

### The "connect once" branch (D-03)
- Shown only when there is **no stored identity AND no network** on a first-ever launch. It is a
  calm static screen (heading + body + a passive offline dot), never a crash or an infinite spinner.
  When connectivity returns, it transitions to the name-picker (reuse `useOnlineStatus()`).

### Connection status = the existing SyncDot vocabulary (D-07, extend — do NOT fork)
| State | Glyph | Meaning this phase |
|-------|-------|--------------------|
| Connected/online | filled `#22C55E` | Session fresh / token valid / online |
| Reconnecting (AUTH-08) | amber `#F59E0B` (the existing drift-amber slot's calm language) | Stale/expired token or offline gap — refresh pending; **never** "logged out" |
| Offline | hollow `#a1a1aa` ring | No network; identity still present, dex fully usable |

Add a "reconnecting/stale-token" read to `SyncDot` reusing its existing 3-state glyph language +
distinct `aria-label`. No second connection indicator anywhere (Research Anti-Pattern).

### Motion timings (Claude's discretion, reduced-motion-aware)
| Element | Motion (reduced-motion fallback) |
|---------|----------------------------------|
| Identity sheet open/close (D-14) | slide-up + scrim cross-fade, ~250ms (reduced: instant show/hide, scrim opacity only) — reuse the app's shipped `<Sheet>` animation, do not invent a new one |
| Roster name tap → password reveal (D-01) | ~150ms field focus/slide (reduced: instant) |
| Sign-in → app (D-17) | instant route switch (no bespoke transition — matches the synchronous route tree) |
| Sign-out → sign-in (D-10) | instant teardown, no fade (prevents any flash of the prior dex) |
| SyncDot state change | reuse existing SyncDot transition (no new motion) |

### Layering (reuse `config.ui.z` tiers, no scattered `z-NN`)
| Surface | Tier | Behavior |
|---------|------|----------|
| Sign-in screen / "connect once" | Full-screen gate **above** the app tree | The gate replaces `<App/>` entirely when no identity — it is not an overlay; nothing paints behind it |
| Identity bottom-sheet (D-14) | `sheet`/`sheetScrim` (the app's existing top-most sheet tier) | Must be the top-most surface (no FAB/banner paints over it — the app-wide sheet-layering discipline) |
| Header identity avatar (D-14) | in the header chrome beside the menu button (`AppShell.tsx:44-53`) | Static chrome glyph; tap opens the sheet |

## Accessibility

- **Every interactive element is a labeled, focusable control** (roster buttons, password field,
  submit CTA, header avatar, sign-out) with an accent `#f2c14e` focus ring — matches the Phase-8
  labeling discipline.
- **Identity never relies on color alone** — the avatar always renders initials; connection status
  always carries an `aria-label` (extends the existing SyncDot `aria-label`).
- **Password input ≥16px** to prevent iOS focus-zoom (venue one-thumb entry).
- **Generic auth error** (no user enumeration) is both a security control (ASVS V7) and a
  clarity/UX control.
- **Reduced-motion** path is first-class for the sheet + backdrop (reuse `useReducedMotion()`).
- No ARIA live-region auto-announcements required (proportionate for a <10-user personal tool; the
  owner is not a screen-reader user — matches the Phase-16 precedent).

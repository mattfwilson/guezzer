# Phase 3 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Foundational phase.** This is the first UI phase. The tokens declared here (spacing, type, color, dark theme, 44px tap floor) are the design foundation every later phase (4 Show Mode, 6 Pokédex, 7 Constellation) inherits. Define once, reuse everywhere — do not re-derive tokens downstream.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (raw Tailwind v4 utilities) |
| Icon library | `lucide-react` (tree-shaken, bundled for offline) — see iOS Share glyph exception below |
| Font | System font stack (no web-font download → faster first paint, better offline) |

**Why `Tool: none` (shadcn gate outcome):** `components.json` is absent and the stack is React/Vite, but `CLAUDE.md` authoritatively locks styling to **Tailwind CSS 4.3.2 via `@tailwindcss/vite`** with an explicit "no component library, no routing library" posture and a hard `MUST follow exactly` override, on a 6–8 week single-developer deadline. The Phase 3 surface is a handful of primitives (tab bar, banner, toast, menu sheet, placeholder cards). Introducing shadcn/Radix would contradict the locked stack for negligible benefit. Registry safety gate: **not applicable** (no registry).

**Font stack (exact):**
`system-ui, -apple-system, "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif`
Mixed iOS/Android target → the native system face is the right call; zero download keeps the offline-first guarantee clean.

**Icon library:** `lucide-react` for chrome icons (nav-tab glyphs, menu, close/dismiss, external-link). **Exception — iOS Share glyph (D-04):** the manual iOS install instructions MUST render an accurate replica of Apple's Share icon (rounded square with an up-arrow), not lucide's generic `Share`/`Share2`. iOS users identify the button by its exact shape, not its name. Ship this as a dedicated inline SVG component (`IosShareGlyph`), currentColor-driven.

## Spacing Scale

Declared values (multiples of 4, Tailwind v4 default 4px base):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, inline padding |
| sm | 8px | Compact element spacing, chip padding |
| md | 16px | Default element spacing, card padding, screen gutters |
| lg | 24px | Section padding, banner internal padding |
| xl | 32px | Layout gaps, placeholder-view vertical rhythm |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level / empty-state top spacing |

**Exceptions:**
- **44px minimum interactive target** — hard floor for every tappable element (nav tabs, banner buttons, toast Refresh, menu rows, dismiss/close). This is a functional venue constraint (SHOW-13/PROJECT.md: one-thumb, dark, crowded, possibly-drunk operator), not a spacing token. 44px is a target *hit-area* minimum, not an 8-grid value; apply via `min-h-11 min-w-11` (44px) plus padding, never by shrinking below it.
- Screen-edge safe-area insets use `env(safe-area-inset-*)` (notch/home-indicator) on top of the gutter — additive, not a fixed token.

## Typography

Two weights only: **regular 400** and **semibold 600**. Four sizes.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px | 400 | 1.5 | Paragraph copy, banner/toast body, iOS instruction steps |
| Label | 14px | 600 | 1.4 | Nav-tab labels, button text, menu rows, version stamp |
| Heading | 20px | 600 | 1.2 | Placeholder-view titles, menu section headers, banner headline |
| Display | 28px | 600 | 1.2 | App name in menu/about; largest single moment only |

Notes:
- Body never below 16px (iOS zooms `<16px` form text; also readability in the dark). Version stamp is the one deliberately-quiet element at 14px/monospace-optional but stays ≥ 14px.
- Version stamp string is data, not a type role — render at Label size; a `tabular-nums`/monospace treatment for the SHA segment is acceptable and encouraged for scan-ability.

## Color

Dark theme is **mandatory** (not a user toggle in Phase 3) — the venue is dark and the tool must not blast a bright screen. Values below are the dark palette; there is no light theme this phase.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0C0C10` | App background, all base surfaces |
| Secondary (30%) | `#17171F` | Elevated surfaces: bottom tab bar, menu sheet, banners, toasts, placeholder cards |
| Accent (10%) | `#F2C14E` (warm gold) | See reserved list |
| Destructive | `#EF4444` | Destructive actions only — **unused in Phase 3** (no destructive actions in the shell); declared for downstream consistency |

Supporting neutrals (not part of the 60/30/10 split — text/border scaffolding):

| Role | Value | Usage |
|------|-------|-------|
| Text primary | `#F5F5F7` | Headings, body on dark surfaces |
| Text muted | `#A1A1AA` | Secondary copy, version stamp, dismissed-state hints, placeholder "coming in Phase N" |
| Border / hairline | `#2A2A34` | Card/tab-bar separators, focus-ring base |

**Accent reserved for (explicit list — never "all interactive elements"):**
1. The active nav-tab indicator (icon + label tint + underline/pill) — one tab at a time.
2. Primary CTA fills: the **Install** button (banner + menu entry) and the update-toast **Refresh** button.
3. Focus-visible ring on keyboard/AT focus.

Everything else interactive (dismiss/close, inactive tabs, menu rows, links) uses text-primary or text-muted on the secondary surface — **not** accent. Accent scarcity is what makes "Install" and "Refresh" read as the one thing to tap.

**Manifest colors (PWA-01):** `theme_color: #0C0C10`, `background_color: #0C0C10` — the splash/status-bar chrome matches the dominant surface so install → launch has no white flash.

## Copywriting Contract

Voice: plain, confident, a touch of fan warmth; never jargon-scary. "Refresh," not "Reload service worker."

| Element | Copy |
|---------|------|
| Primary CTA (install) | **Install Guezzer** |
| Primary CTA (update) | **Refresh** |
| Empty state heading (per placeholder view) | Show → **"Show Mode"**; Explore → **"Explore"**; Dex → **"Your Pokédex"** |
| Empty state body (per view) | Show → "The live prediction loop lands in Phase 4."; Explore → "The song constellation lands in Phase 7."; Dex → "Your caught-live collection lands in Phase 6." |
| Install banner headline | **Install Guezzer** |
| Install banner body | "Add it to your home screen so it works offline at the show." |
| Install banner dismiss | **Not now** (re-shows next launch until installed, per D-05; permanent menu entry is the always-on fallback) |
| iOS manual-install heading | **Add Guezzer to your Home Screen** |
| iOS manual-install steps | 1. "Tap the Share button" (with the `IosShareGlyph`) → 2. "Choose **Add to Home Screen**" → 3. "Tap **Add**" |
| Update toast | **New version available — Refresh** |
| Update toast dismiss | **Later** (keeps current version running indefinitely; no auto-swap — D-06) |
| Error / fallback state | Install unavailable in this browser → show the manual-install path instead of an error. Exact copy: **"Guezzer can't auto-install here — add it from your browser menu instead."** `storage.persist()` denial is **silent** (D-09): no error UI. |
| Destructive confirmation | **None** — Phase 3 has no destructive actions. (First destructive flows — undo/edit a logged song, import-overwrite — arrive in Phases 4/5 and will be specified there.) |
| Version stamp (menu/about) | Format: `v{pkgVersion} · {shortSha} · built {YYYY-MM-DD}` — e.g. `v1.0.0 · a1b2c3d · built 2026-08-15` (D-07). Injected at build via Vite `define`. |

## Component Inventory (Phase 3 surface)

Prescriptive list for the planner/executor. All raw Tailwind + lucide; no library components.

| Component | Purpose | Key contract |
|-----------|---------|--------------|
| `AppShell` | Root chrome: header + `<main>` + bottom tab bar | Full-viewport, dark, safe-area aware; `<main>` scrolls, tab bar fixed |
| `BottomTabBar` | Show / Explore / Dex switch | 3 tabs, each ≥ 44px, icon (14–24px) + Label; active tab = accent indicator; drives hash route |
| `PlaceholderView` | Empty Show/Explore/Dex screens | Centered heading (20/600) + muted body; generous 3xl top spacing; no accent |
| `AppMenu` | Menu/about sheet | Houses permanent **Install** entry, version stamp, (future) settings; secondary surface; rows ≥ 44px |
| `InstallBanner` | Dismissible install invite | Secondary surface, accent Install button + muted "Not now"; branches iOS vs Android |
| `IosInstallInstructions` | Illustrated manual steps | Shown only to detected iOS Safari; numbered steps + `IosShareGlyph` |
| `UpdateToast` | SW-waiting update prompt | Non-blocking, bottom, above tab bar; accent Refresh + muted Later; dismiss ≠ apply |
| `IosShareGlyph` | Accurate Apple Share icon | Inline SVG, `currentColor`, not a lucide substitute |

## Layout & Interaction Contract

- **Navigation placement:** bottom **tab bar** (not a top nav) — one-thumb reachability is the governing constraint. Header carries only the app name/wordmark and a menu button (top-right). Tabs map to hash routes `#/show` (default), `#/explore`, `#/dex` (D-02).
- **Routing:** hash routing, no library. Back button works; unknown/empty hash → `#/show`. Shareable-URL PWA property satisfied.
- **Install banner behavior:** on Android/Chrome, gated by a captured `beforeinstallprompt`; on iOS Safari, reveals `IosInstallInstructions`; hidden entirely when already installed (`display-mode: standalone` / `navigator.standalone`). Re-shows next launch after "Not now" until installed (D-05).
- **Update toast behavior:** appears only when a SW is waiting; **applies only on Refresh tap** (`skipWaiting` + reload via `workbox-window`). Never auto-applies — this is the concrete "never swaps mid-show" mechanism (D-06). Non-blocking: the app stays fully usable if ignored.
- **Persistence UI (D-09):** `navigator.storage.persist()` requested early on first run; **no visible storage-status indicator** this phase. Denial is silent; status is recorded to the meta table for a later export-nudge.
- **Motion:** CSS transitions only (banner/toast slide + fade, tab active-state). No `motion` lib in Phase 3. Respect `prefers-reduced-motion` (reduce to opacity/instant).
- **Contrast / a11y floor:** all text ≥ 4.5:1 against its surface; accent-on-dark CTA text uses `#0C0C10` foreground on the gold fill (dark-on-gold) to clear 4.5:1. Focus-visible ring on all interactive elements.

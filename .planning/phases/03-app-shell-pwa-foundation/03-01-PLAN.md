<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| build → bundle | Build-time values (git SHA, build date, package version) are injected into the client bundle via Vite `define`. |
| network → service worker cache | The Workbox SW precaches the app shell; on reload it serves cached assets (the offline path). |
| URL fragment → app state | `location.hash` (user/attacker-controllable) selects which view renders. |
| npm registry → dependency tree | This plan performs the phase's dependency installs. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | Service worker / precache | mitigate | SW is Workbox-generated, same-origin, served over HTTPS (SW requires it); no hand-written `sw.js`, no third-party SW. `registerType:'prompt'` means nothing auto-activates. |
| T-03-02 | Tampering/EoP | Hash router (`useHashRoute`) | mitigate | `location.hash` is validated against the fixed `ROUTES` allow-list and only used to select a React view — never assigned to `innerHTML`, `location`, or passed to `eval`. |
| T-03-03 | Tampering | XSS via injected build constants | mitigate | `__APP_VERSION__`/`__GIT_SHA__`/`__BUILD_DATE__` are build-time string literals; rendered only as text (Plan 03), never as HTML. |
| T-03-SC | Tampering | npm installs (react, dexie, vite-plugin-pwa, tailwind, testing libs, fake-indexeddb) | mitigate | All packages covered by RESEARCH §Package Legitimacy Audit (17 OK; `vitest` SUS = documented typosquat-vs-vite false positive, already a root dep). `fake-indexeddb` (the one [ASSUMED] item) verified at plan time: Apache-2.0, github.com/dumbmatter/fakeIndexedDB, created 2015, v6.2.5 — legit. No unresolved [ASSUMED]/[SUS]/[SLOP] packages remain, so no blocking checkpoint is required. |
| T-03-04 | Denial of Service | White-flash / broken install | accept | Manifest `theme_color`/`background_color` set to `#0C0C10` (Pitfall 9) to avoid a bright flash; cosmetic, low severity. |

No `security_block_on: high` items. Surface is LOW (no backend, no auth, no PII) per RESEARCH §Security Domain.
</threat_model>

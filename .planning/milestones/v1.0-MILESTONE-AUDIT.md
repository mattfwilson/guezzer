### ⚠ WARNING-1 — Own-backup restore misroutes to friend-compare on a fresh/evicted DB
`packages/app/src/settings/importPicker.ts:66-74`: `classifyImport` returns `"friend"` when a backup has an `owner` but the **local** owner name is null. On a genuine iOS-eviction reinstall — exactly the scenario PWA-04 exists for — the local owner meta row is evicted too, so importing your own named backup opens the **read-only CompareView** (zero writes) instead of restoring. **Non-destructive and recoverable** (set owner name to match, then re-import → merges), so WARNING not BLOCKER — but it undercuts the headline phone-loss recovery flow. Suggested fix: prompt "Whose dex is this?" for an owned file when local owner is null, or add a "restore as mine" affordance on the compare screen. Affects PWA-04, DEX-04.

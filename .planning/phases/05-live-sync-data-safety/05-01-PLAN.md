<objective>
Build the pure, zero-DOM live-sync core for Phase 5: a latest-endpoint-specific zod schema, a tolerant `latest` poller, the dedupe-to-suggestions diff, the "???" placeholder resolver, and the wrong-show-guarded auto-bind decision. All logic is dependency-injected and Node-testable, mirroring the existing ingestion layer, so the app tier (Plan 04) only owns lifecycle/timing.

This plan is the enabling substrate for the live-sync vertical slice whose user story is: *As a show-tracker, I want editor-logged songs to appear as deduped suggestions during a show, so that I get a second set of eyes without ever clobbering my manual tracking (SYNC-01, SYNC-02).*

Purpose: SYNC-01 (poll `latest` only, validate rows, enforce artist_id) and SYNC-02 (suggestions-only dedupe) live in core per D-05, keeping strict core/UI separation (CLAUDE.md).
Output: `latest-types.ts`, `live/poll-latest.ts`, `live/suggest.ts`, `live/bind-show.ts`, a committed `data/samples/latest.sample.json`, barrel exports, and four Node test files.
</objective>

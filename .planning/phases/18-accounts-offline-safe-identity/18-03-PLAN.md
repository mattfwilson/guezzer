<objective>
Build the app-owned identity record — the substrate that makes the offline-safe boot gate architecturally immune to the `getSession()`-returns-null-offline bug (RESEARCH Pitfall 1 / Pattern 2, the belt-and-suspenders reading of D-06). It is a tiny persistent `{ userId, displayName }` store on a dedicated localStorage key, read synchronously (zero await, D-05), plus a `useSyncExternalStore`-based hook so sign-in and sign-out re-render subscribers. This has no Supabase or UI dependency, so it runs in Wave 1 and unblocks the sign-in surface (Plan 04), the identity chrome (Plan 05), and the gate (Plan 06) to proceed in parallel.

Purpose: The boot gate keys on presence of THIS record, not on token validity or a network call — that is what guarantees a friend whose token expired overnight is never locked out of their own offline dex.
Output: `identityRecord.ts`, `useAuthIdentity.ts`, `identityRecord.test.ts`.
</objective>

# Phase 19 — UI Design Contract

> Visual and interaction contract for the Shared Dex Progress friends surface. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **This phase is "reuse, don't reinvent."** Every token below is the shipped Guezzer design system (18 phases in). The Friends surface introduces **zero new chrome color, no new type role, no new spacing unit, and no new component library.** New components echo shipped ones (`CompareView`, `DexHeader`, `TierBadge`, `IdentityAvatar`, `SyncDot`) without importing/altering them.

## Spacing Scale

Declared values (Tailwind default 4px scale — already the app-wide contract; multiples of 4 only):

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs | 4px (`gap-1`, `py-0.5`) | Badge inner padding, tight stat gaps |
| sm | 8px (`gap-2`, `p-1`) | Row element gaps, segment-track padding |
| md | 12–16px (`gap-3`, `px-4`, `py-3`) | Row padding, list gutters, header padding |
| lg | 24px (`pb-6`) | Section breaks within FriendDetail |
| xl | 32px (`pt-8`) | Overlay top rhythm (mirrors `DexHeader`) |

Exceptions:
- **44px minimum tap target** (`min-h-11 min-w-11`) on every interactive element — friend rows, segment tabs, overlay close/back, expandable breakdown headers. Non-negotiable (SHOW-02 venue "one-thumb-in-the-dark" floor, held app-wide).
- **32px identity glyph** (`h-8 w-8`) — reused verbatim from `IdentityAvatar`; the 44px hit area is the wrapping button, not the glyph.
- **Safe-area top inset** on the FriendDetail overlay header: `calc(env(safe-area-inset-top) + 12px)` (the shipped `CompareView`/sheet header idiom — never a bare inset, never doubled).

## Typography

App-wide roles (unchanged — this phase adds none):

| Role | Size | Weight | Line Height | Usage in this phase |
|------|------|--------|-------------|---------------------|
| Display | 56px | 600 | 1.1 (`leading-tight`) | Optional head-to-head hero completion figure in FriendDetail (reuse of the `DexHeader` idiom); NOT required on list rows |
| Heading | 20px | 600 | 1.2 (`leading-tight`) | FriendDetail overlay title ("You vs {name}"), showcase heading |
| Body | 16px | 400 | 1.5 (`leading-normal`, `text-base`) | Row completion/caught text, stat values, empty-state body, offline marker |
| Label | 14px | 600 | 1.25 (`leading-tight`, `text-[14px]`) | Segment tab label, `TierBadge` word, stat-column headings, breakdown section headers, "caught" caption |

Weights: exactly two — **400 (regular)** and **600 (semibold)**. No other weight anywhere.

Notes:
- A **13px** small-caption variant exists app-wide (avatar initials, sign-out subline). Permitted for the offline-marker timestamp / secondary sublines only — do not introduce it as a new primary role.
- **`tabular-nums`** is mandatory on every completion %, caught count, and per-tier/per-album count so live-updating numbers don't reflow (matches `DexHeader`/`CompareView`).
- Friend-supplied names render **escaped React text, `truncate`/`min-w-0` clamped** (T-06-26 / D-19) — never `dangerouslySetInnerHTML`, never width-unbounded.

## Color

Dark-only palette from `@theme` (`styles.css`). 60/30/10 split unchanged:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0C0C10` (`--color-surface`) | App / Friends list background |
| Secondary (30%) | `#17171F` (`--color-elevated`) | Friend row cards, segment-control track, FriendDetail sheet + stat columns, offline marker chip |
| Accent (10%) | `#F2C14E` (`--color-accent`) | **ONLY** the active `Friends` segment tab fill (`bg-accent text-surface`) — the single accent use this phase adds |
| Destructive | `#EF4444` (`--color-destructive`) | **Unused this phase** — no destructive action exists (write-own only; no delete/reset). Declared, not applied. |
| Text primary | `#F5F5F7` (`--color-text-primary`) | Names, stat values |
| Text muted | `#A1A1AA` (`--color-text-muted`) | Captions, counts, offline marker, empty-state body |
| Hairline | `#2A2A34` (`--color-hairline`) | Row dividers, column grid lines, breakdown borders |

**Accent reserved for:** the active `Friends` segment tab only. No accent buttons, no accent friend-row states, no accent on the FriendDetail overlay. (The dex Share CTA — the only other accent control in GizzDex — is NOT part of the Friends surface.)

**Data-semantic palettes — exempt from the 10% accent budget (B3: "data semantics never chrome"):**

| Palette | Values | Usage — carries meaning, never a button/surface/focus color |
|---------|--------|------------------------------------------------------------|
| Rarity tiers | debut `#A1A1AA` · common `#E4E4E7` · uncommon `#34D399` · rare `#60A5FA` · epic `#A855F7` · legendary `#FB923C` (`config.dex.tierColors`, via `rarityColor`) | The single rarest-tier badge on each row (PROG-04) and the top-5 rarest-catches showcase (PROG-08). **Reuse `TierBadge` verbatim** — the tier WORD always renders alongside the hue (WCAG 1.4.1). On-badge text stays transparent-fill/40%-border per `TierBadge`; showcase-on-fill text uses `RARITY_ORB_TEXT_COLOR` `#0C0C10` (≥4.5:1 verified on all six hues). |
| Identity | `#7DD3FC · #FDA4AF · #86EFAC · #FCD34D · #C4B5FD · #5EEAD4` (`config.auth.IDENTITY_COLORS`, via `identityColorIndex(userId, len)`) | Friend-row + self-row avatar glyph fill — **stable per user across devices** (AUTH-07). Initials always render in `#0C0C10` (identity is never color alone, D-12). Reuse the `IdentityAvatar` glyph pattern. |

These two palettes are the app's shipped "people vs rarities" language — do NOT recolor, do NOT route them through `--color-accent`, do NOT let a friend row adopt an accent surface.

## Copywriting Contract

New copy lives in a `config.copy.friends` block (single-config-file ethos, CLAUDE.md). Head-to-head columns **reuse the shipped `config.copy.compare` strings** (`columnYou`, `statCompletion`, `statCaught`, `statShows`) so the live path and the file-import path never disagree.

| Element | Copy |
|---------|------|
| Segment tab | `Friends` (D-01) |
| Primary interaction | **No accent button CTA.** Entry is the `Friends` segment tab; the primary action is *tap a friend row → live head-to-head*. Row affordance = trailing `ChevronRight`, no label. |
| Self-row label | `You` (D-02, pinned top) |
| Row completion | `{pct}%` (`tabular-nums`) |
| Row caught count | `{n} caught` |
| Zero-catch friend row (D-05) | `0% · 0 caught` — no badge, sorted last. Honest "here, hasn't caught yet," never hidden. |
| Empty state heading | `No friends yet` |
| Empty state body | `When your Gizz friends sign in and start catching songs, they'll show up here.` |
| Offline / stale marker (D-18) | `Offline · as of {time}` — calm, muted chip; reuses the SyncDot connection language. Never a spinner, never blank. |
| Degraded read (online, fetch failed) | `Can't reach friends right now — showing the last sync.` (a malformed *individual* row is silently skipped per D-19 — never a per-row error) |
| FriendDetail overlay back/close | `Back` (aria-label; leading `ChevronLeft`, or `X` matching `CompareView` — executor's choice, must be a ≥44px control) |
| Head-to-head heading | `You vs {name}` |
| Head-to-head columns | Reuse `config.copy.compare`: `You` / `{name}`, `Completion`, `Caught`, `Shows` |
| Per-album breakdown heading | `By album` |
| Per-rarity breakdown heading | `By rarity` |
| Rarest-catches showcase (friend) | `{name}'s rarest catches` (PROG-08) |
| Rarest-catches showcase (own "You" trophy case, D-06) | `Your rarest catches` |
| Showcase empty (0-catch friend detail) | `No catches yet` |
| Error state | Same as "Degraded read" above — connection-language, reassuring, never alarming |
| Destructive confirmation | **None** — this phase has no destructive action (write-own RLS; no delete, no reset, no inflation path) |

All friend-supplied strings (`display_name`, any song name) render as **escaped React text, clamped** (D-19 / T-06-26). Songs are identified by `songId` only for all set arithmetic (duplicate names in the matrix).

## Interaction & States

| State | Behavior |
|-------|----------|
| First sync (online, no cache) | Pinned `You` row renders **immediately** from local dex; friend area holds a calm frame (no spinner, no NaN) until the first pull resolves. |
| Populated (online) | `You` pinned top; friends below, completion-desc. Live via `postgres_changes` app-wide subscription (D-16). |
| Live update (PROG-05) | A friend's catch → full ~5-row re-pull → rows update and re-order. **Reduced-motion-aware**: a calm value/position change ("progress visibly moves"), never a jarring flash or celebratory burst. Reuse existing calm-transition idioms; add no new keyframes. |
| Offline / dead-signal venue (D-18) | Show **last-known cached** friend rows, **dimmed**, with the `Offline · as of {time}` marker. The `You` row stays **live** (local dex). Friends are **never blank** at the venue. |
| Reconnect (D-17) | Flush own row + re-pull all friends once; drop the offline marker; un-dim rows. Reuse `useOnlineStatus` / SyncDot signal. |
| Zero-catch friend (D-05) | `0% · 0 caught`, no badge, sorted last. |
| Tap a friend | Opens `FriendDetail` full-screen overlay: **leads** with "You vs {name}" head-to-head columns, then the expandable per-album / per-rarity breakdown, then their rarest-catches showcase (D-08). |
| Tap `You` (D-06) | Opens the own trophy-case showcase — rarest catches only, **no head-to-head columns** (no self-compare). |
| Malformed friend row (D-19) | zod-validate at the read boundary; a bad row degrades to skipped, never crashes the list. |

## Layering

FriendDetail overlay uses the shipped **`sheet` z-tier** (`config.ui.z.sheet` = 50) via `Sheet variant="fullscreen"`, exactly like `CompareView`. **No new z-index tier** — the app's named z-scale (`config.ui.z`) is the single source of truth. The FriendDetail sits above the bottom tab bar and any FAB, below toasts is not a concern (there is no persistent toast on this surface this phase).

## Accessibility

- Every interactive element ≥44px hit area (`min-h-11 min-w-11`).
- Rarity conveyed by **word + color** (`TierBadge` always renders the tier word — WCAG 1.4.1).
- Identity conveyed by **initials + color** (never color alone — D-12).
- `tabular-nums` on all live-updating figures (no reflow on update).
- Friend names: escaped React text, `truncate` clamped, `aria` labels on icon-only controls (back/close, row affordance).
- Live row updates announced calmly; honor `prefers-reduced-motion` on any reorder/transition.
- Overlay is a modal `Sheet` with a labeled back/close affordance and a focus trap (shipped `Sheet` behavior).

# Phase 8 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Hardening phase — no new user-facing features.** This phase adds ZERO new visual surface. It inherits every token (spacing, typography, color, dark theme, 44px tap floor) from **Phase 3's foundational UI-SPEC** and does not re-derive them. What it *does* add is a small set of **interaction contracts** (focus management, Escape-dismiss, z-index layering, FAB-lift choreography, resize reframe) and one **legibility retune** (orb-label fit floor). Those are the load-bearing new contracts specified below; the token tables are reproduced only for the checker's reference and are marked as inherited.

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (raw Tailwind v4 utilities) — **CLAUDE.md locks styling to Tailwind CSS 4.3.2 via `@tailwindcss/vite`, no component library, no routing library** |
| Icon library | `lucide-react` 1.23.0 (tree-shaken, bundled offline); iOS Share glyph is a dedicated inline SVG (`IosShareGlyph`) |
| Font | System font stack — `system-ui, -apple-system, "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif` (no web-font download; offline-first) |

**shadcn gate outcome:** `components.json` is absent and the stack is React/Vite, but the shadcn gate is **already resolved as `Tool: none`** by Phase 3's UI-SPEC and re-affirmed here. CLAUDE.md carries a hard `MUST follow exactly` override locking the stack to raw Tailwind v4 with an explicit "no component library" posture, and this is a tight, pre-show *hardening* pass. Introducing shadcn/Radix now would contradict the locked stack, expand surface area against the milestone's "no new features / low-risk" bias, and fight the app's inline `fixed inset-0` sheet idiom (see RESEARCH §Alternatives — native `<dialog>`/`focus-trap-react` both rejected). **Registry safety gate: not applicable (no registry).**

**Net-new npm dependencies this phase: ZERO.** The a11y layer is ~120 lines of dependency-free hooks built on React 19's native `inert` boolean prop + baseline DOM APIs (RESEARCH §Standard Stack). Optional dev-only `@testing-library/user-event` is gated behind `checkpoint:human-verify` if the planner wants real Tab-wrap assertions.

## Spacing Scale

**Inherited verbatim from Phase 3 UI-SPEC** (Tailwind v4 4px base). Reproduced for the checker; not re-derived.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, inline padding |
| sm | 8px | Compact element spacing, chip padding |
| md | 16px | Default element spacing, card padding, screen gutters |
| lg | 24px | Section padding, sheet internal padding |
| xl | 32px | Layout gaps, sheet safe-area bottom padding base |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level / empty-state top spacing |

**Exceptions:**
- **44px minimum interactive target** — hard floor for every tappable element (venue constraint: one-thumb, dark, crowded). Applied via `min-h-11 min-w-11` + padding. Every control this phase touches (FAB, sheet close buttons, dialog actions) preserves this floor.
- **`FAB_SHEET_GAP_PX = 12` (NEW, multiple of 4)** — the gap the lifted Explore FilterFab leaves above the NodeSheet's top edge (D-03 / A11Y-02). Lands in `config.ui`. 12px = the `sm`+`xs` composite, on-grid.
- Screen-edge safe-area insets use `env(safe-area-inset-*)` additively (not a fixed token).

## Typography

**Inherited verbatim from Phase 3 UI-SPEC** — two weights (regular 400 / semibold 600), four sizes. Reproduced for the checker.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px | 400 | 1.5 | Paragraph copy, sheet body, dialog body |
| Label | 14px | 600 | 1.4 | Button text, menu rows, chips |
| Heading | 20px | 600 | 1.2 | Sheet/dialog titles |
| Display | 28px | 600 | 1.2 | Largest single-moment text (dex completion, recap hero) |

**NEW exception — orb-label data-text floor (POLISH-01 / D-05):**

Orb/center-node song labels are **canvas/SVG data-viz text on a colored orb**, a separate axis from the body-copy sizes above (they are keyed off `config.show.ORB_LABEL_*`, not the type roles). This phase lowers the fit floor:

| Constant | Current | **Retune (recommended)** | Rationale |
|----------|---------|--------------------------|-----------|
| `ORB_LABEL_MAX_LINES` | 3 | **4** | Clears every real catalog name at any orb ≥64px (RESEARCH simulation over the 264-node artifact). |
| `ORB_LABEL_MIN_FONT_PX` | 11 | **10** | Documented **minimum-legibility floor for 600-weight orb data-text in the dark**. Below 10px is not legible at a show. |
| `ORB_LABEL_MAX_LINES_CENTER` | 3 | 3 (verify) | Center variant fits against `diameter − 24`; bump only if the on-device harness shows overflow. |
| `ORB_LABEL_MIN_FONT_PX_CENTER` | 12 | 12 (verify) | Larger center orb; keep unless the 44-char outlier ellipsizes at real size. |
| `CHAR_WIDTH_FACTOR` | 0.52 | **measure on-device, likely raise to ~0.55–0.58** | The heuristic drifts *optimistic* (bold advance, circular chord, `USABLE_WIDTH_FACTOR=1`); make it conservative, then compensate with the extra line / lower floor. |

**Legibility floor declaration:** **10px @ weight 600** is the absolute minimum for a prediction-orb/center-node label. The ellipsis last-line fallback stays only as a theoretical guard that must NOT fire for any real catalog name at that name's real rendered diameter. **Verification bar is the actual 264-song catalog rendered on real hardware** (the dev harness on the cloudflared UAT tunnel), never the no-DOM heuristic's self-report (D-05, RESEARCH §Orb-Label Legibility).

## Color

**Inherited verbatim from Phase 3 UI-SPEC** — dark theme mandatory, no new colors introduced this phase. Values are the CSS `@theme` tokens in `styles.css`.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0C0C10` (`--color-surface`) | App background, all base surfaces, modal backdrops (at 50% opacity) |
| Secondary (30%) | `#17171F` (`--color-elevated`) | Sheets, dialogs, tab bar, FAB, banners |
| Accent (10%) | `#F2C14E` (`--color-accent`) | See reserved list |
| Destructive | `#EF4444` (`--color-destructive`) | Destructive confirmations only (EndShowDialog, unmark) — unchanged this phase |

Supporting neutrals (text/border scaffolding, not part of 60/30/10): text primary `#F5F5F7`, text muted `#A1A1AA`, hairline/focus-ring base `#2A2A34`.

**Accent reserved for (unchanged from Phase 3 — this phase adds nothing to the list):**
1. Active nav-tab indicator (one tab at a time).
2. Primary CTA fills (Install, Refresh, Start Show, Export backup, Share card, the pre-opener prompt orb).
3. **Focus-visible ring on keyboard/AT focus** — *this is the accent element this phase exercises most heavily.* Every control that gains focus management (all 7 sheets/dialogs + the lifted FilterFab) MUST show a visible `#F2C14E` focus-visible ring when reached via keyboard/AT. This is the a11y-visible manifestation of A11Y-01 and must clear 4.5:1 against its surface.

**No new hue, tint, or semantic color is introduced.** The z-index tier scale is a stacking contract, not a color contract; the FAB lift is a transform, not a recolor.

## Layering (z-index) Contract — NEW (D-04)

The single most important **new visual contract** of this phase: a centralized, named z-index tier scale in the single config file (`config.ui.z`), replacing all scattered raw `z-NN` Tailwind literals. Applied via inline `style={{ zIndex: config.ui.z.X }}` on each `fixed` overlay (Tailwind v4 resolves arbitrary `z-[...]` at class-author time, so a JS-config value must go through inline style — keeps the single source in config.ts per CLAUDE.md).

| Tier | Value | Occupants |
|------|-------|-----------|
| `content` | 10 | In-flow raised content (ShowView column) |
| `toast` | 20 | UpdateToast, InstallBanner |
| `fab` | 30 | Resting FABs + FabMenu action rows |
| `fabScrim` | 35 | FabMenu full-viewport scrim (below sheets, above content) |
| `sheetScrim` | 40 | Modal sheet/dialog backdrops |
| `sheet` | 50 | Sheet/dialog content (all 6 modals + the non-modal NodeSheet) |
| `focusedFab` | 60 | **D-03 deliberate exception:** the Explore FilterFab lifted above the NodeSheet when a constellation node is focused |

**Invariant (HIGH confidence):** every FAB tier sits strictly below `sheet` by default; `focusedFab` is the ONE sanctioned exception. **Migrate ALL current `z-*` usages** (RESEARCH found 24, more than CONTEXT.md's ~8 — including `AlbumDetail`, `CometTrail`, `ArchiveBrowser`, `SearchSheet`, `RecapView`, `SetlistView`, `ShowView`) onto tiers so no `z-NN` literal remains and the fix cannot relocate a collision. The specific integers are `[ASSUMED]` pending the planner's explicit 24-usage map; the *ordering* invariants are the contract.

## Interaction Contract — Sheets & Dialogs (NEW, A11Y-01/02/03)

The behavioral heart of this phase. One shared `<Sheet>` primitive + `useFocusTrap` + `useDialogDismiss` hooks (D-01), with a deliberate modal / non-modal split (D-02).

### Focus & dismiss behavior per surface

| Surface | Treatment | Escape | Focus trap | Focus restore | `inert` bg | Scrim |
|---------|-----------|--------|-----------|---------------|-----------|-------|
| AppMenu | full modal | ✓ | ✓ | ✓ | ✓ | ✓ |
| TrailNodeSheet | full modal (stacks → SearchSheet) | ✓ (topmost only) | ✓ | ✓ | ✓ (ref-counted) | ✓ |
| EndShowDialog | full modal | ✓ | ✓ | ✓ | ✓ | ✓ |
| ShareCardSheet | full modal | ✓ | ✓ | ✓ | ✓ | ✓ |
| CompareView | full modal, **fullscreen** variant (header X, no backdrop) | ✓ | ✓ | ✓ | ✓ | ✗ (fullscreen) |
| "Whose dex is this?" prompt | full modal; text input gets `initialFocusRef`/autofocus | ✓ | ✓ | ✓ | ✓ | ✓ |
| WhyDetail | full modal — **RESEARCH recommends inclusion** (cheapest win, same idiom); **confirm at planning** | ✓ | ✓ | ✓ | ✓ | ✓ |
| **NodeSheet** (D-02 special case) | **non-modal** — graph stays live/interactive above it | ✓ | ✗ (none) | ✓ | ✗ (none) | ✗ (none) |

**Contract rules:**
- **Escape dismisses exactly one (topmost) dialog** via a shared module-level LIFO `dialogStack` + one global keydown listener — never a per-dialog `document` listener (those double-fire across the app's stacked dialogs and would collapse the whole stack).
- **Focus restores to the triggering element** (captured `document.activeElement` on open) when any surface closes.
- **`inert` on the app-content root** is the true background suppression for the 6+1 modals (React 19 native boolean prop), ref-counted so stacked modals compose correctly. `aria-modal="true"` is kept as an AT hint but is explicitly **NOT** treated as a focus trap (it never was — the current bug).
- **NodeSheet stays a live window on the graph** — no scrim, no trap, no `inert`; focus must still reach the constellation and the FilterFab. It gets Escape + focus-restore only.
- **No guarded error state may become a throw** through the migration — `<Sheet>` renders children as-is; keep existing `if (!open) return null` guards.

### FilterFab lift choreography (A11Y-02 / D-03)

- While a constellation node is focused, the Explore FilterFab **lifts via `translateY`** to rest `FAB_SHEET_GAP_PX` (12px) above the NodeSheet's top edge — fully visible and tappable, no overlap — and returns to its resting bottom-right position when the sheet closes.
- Lift distance is computed from a **shared visible-viewport height source** (`window.visualViewport?.height ?? window.innerHeight` or the container ResizeObserver), NOT `window.innerHeight` alone — the latter is the iOS large viewport and mismatches the constellation container (latent bug in `NodeSheet.tsx:82`). The same source feeds the NodeSheet peek and the camera reframe so they never disagree.
- Lifted FAB uses `z.focusedFab` (60); resting uses `z.fab` (30).

### Resize reframe (A11Y-03)

- Resizing the viewport (orientation change, iOS keyboard show/hide, address-bar collapse) with a node focused **keeps the camera framed on that node** — no snap-off. The focus-camera effect already lists `size.height` in deps (`ConstellationCanvas.tsx:315`); treat as verify-on-device + fix edge cases, not a from-scratch build.

### Motion contract

- **All new motion honors `prefers-reduced-motion`.** The FAB-lift transition, and any sheet transition, become **instant** (`transition: none`) under reduced-motion — matching the app-wide `motion-safe:` idiom and the existing `@media (prefers-reduced-motion: no-preference)` gating in `styles.css`. (The sheet slide-up/down + scrim cross-fade *animation* itself is DEFERRED to a later polish pass — this phase ships the layering + a11y only.)

## Copywriting Contract

**This phase introduces no new visible body copy.** All CTA / empty / error / destructive strings are inherited verbatim from `config.copy` (Phases 3–7). The new contract is **accessible names** (the a11y equivalent of copywriting) — every dialog surfaced by A11Y-01 must carry an accessible name so screen-reader users hear what opened.

| Element | Copy / accessible name | Source |
|---------|------------------------|--------|
| Primary CTA (unchanged) | inherited per-view (Start Show, Export backup, Share card, Refresh, Install Guezzer) | `config.copy.*` |
| Empty state | inherited per-view (`dex.emptyHeading`/`Body`, `explore.rotationEmptyHeading`, etc.) | `config.copy.*` |
| Error state | inherited (`show.modelLoadFailureHeading`, `explore.errorHeading`, `share.failureHeading`) | `config.copy.*` |
| Destructive confirmation | inherited (`show.endHeading`/`endConfirm`, `archive.unmarkHeading`/`unmarkConfirm`) — unchanged | `config.copy.*` |
| **Sheet/dialog accessible names (NEW)** | Each `<Sheet ariaLabel>` reuses the surface's existing heading string — AppMenu → Settings/menu label; TrailNodeSheet → the trail entry; EndShowDialog → `show.endHeading` ("End show?"); ShareCardSheet → `share.sheetLabel` ("Share card"); CompareView → `compare.banner`; name prompt → `compare.namePrompt` ("Whose dex is this?"); NodeSheet → the focused song name; WhyDetail → the orb's song name + "why". | reuse `config.copy.*` — **no new strings, no duplication** |
| FilterFab accessible label | `config.copy.explore.filterFabAria` ("Explore filters") — unchanged | `config.copy.explore` |

**Rule:** the `<Sheet>` primitive takes an `ariaLabel` prop wired from the existing heading string — **do not author a second copy of any label.** Song/venue names remain kglw-derived React text / canvas `fillText` only (never `dangerouslySetInnerHTML`).

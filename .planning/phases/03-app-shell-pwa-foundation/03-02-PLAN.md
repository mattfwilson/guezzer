<objective>
Deliver the install-onboarding vertical slice of PWA-01: turn "get friends to actually install" into a real, working invitation. Add a dismissible install banner that branches by platform (Android `beforeinstallprompt` prompt vs. detected-iOS-Safari illustrated manual instructions), a permanent Install entry in the app menu as the always-available fallback, and the re-show-until-installed behavior.

Purpose: The install banner is the deliberate mitigation for the iOS Safari IndexedDB-eviction risk called out repeatedly in PROJECT.md â€” treat it as a functional requirement, not polish. iOS Safari never fires `beforeinstallprompt`, so iOS users need the illustrated Share-icon path or they have no way to install (Pitfall 2/3).

Output: `platform.ts` detection primitives, the `useInstallState` hook, `InstallBanner` / `IosInstallInstructions` / `IosShareGlyph` components, an `AppMenu` sheet with the permanent Install entry, the banner+menu wired into `App.tsx`, and a `platform.test.ts` unit test.

Covers PWA-01 (install onboarding, including manual iOS instructions). Implements D-03 (dismissible banner + permanent menu entry), D-04 (illustrated iOS instructions with the real Apple Share glyph), D-05 (re-show next launch until installed; session-only dismissal).
</objective>

<phase_goal>
**As a** friend attending a King Gizzard show, **I want to** install Guezzer to my home screen and have it load fully offline with no surprise version swaps or data loss, **so that** I can rely on it in a dark venue with no signal.

This plan delivers the install-invitation slice: after it completes, a non-installed visitor is guided to add Guezzer to their home screen on both iOS and Android.
</phase_goal>

## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Testing against a stale dist | `vite preview` serves whatever is on disk in `packages/app/dist` — it was 2 days older than source, so a "missing" feature (SyncDot) was chased as a code bug. Browser cache-clearing cannot fix a stale build. | advisory | Always `npm run build` in packages/app before any preview-based manual test; sanity-grep the served bundle for a marker string when in doubt. |
| Diagnosing device-only bugs from static code reads | Two rounds of plausible-but-wrong layout fixes (banner overlap, iOS 100vh) shipped before instrumentation revealed the real cause (crypto.randomUUID missing on insecure origins). | advisory | For works-on-desktop/fails-on-device bugs, instrument first (on-screen error/tap reporter or automated browser drive) — get one piece of ground-truth device evidence before writing any fix. |

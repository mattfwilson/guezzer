<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| committed JSON -> review CLI | Local already-committed corpus/albums/tags read into a Node process; no network, no user input |
| owner hand-edit -> build-model reload (D-03 branch only) | A hand-authored tuning-tags.json edit re-enters the model pipeline |
| markdown report -> viewer (GitHub/VS Code preview) | Catalog-sourced song names embedded in generated markdown |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | Tampering | review-tuning-tags.ts markdown report (data/tuning-review.md) | mitigate | Escape every catalog-sourced song name with copied `escapeMarkdownExcerpt` (&/</>) before embedding — markdown-viewers render embedded HTML |
| T-10-02 | Tampering | data/tuning-tags.json hand-edit (D-03 branch) | mitigate | `tuningTagsFileSchema.parse` on next build-model load hard-fails a corrupt/duplicate/out-of-vocabulary edit; the D-03 backtest compare catches a semantic regression |
| T-10-03 | Information disclosure | read-only review CLI | accept | Reads only already-committed local JSON; no network, no secrets, writes only the local review report |
| T-10-SC | Tampering | npm/pip/cargo installs | accept | This plan installs NO new packages (all deps present); legitimacy gate N/A |
</threat_model>

<objective>
SAFE-04: two shows attended on the same date must survive as two distinct attendances
through BOTH `merge.ts` and `derive-dex.ts`. Root cause: a duplicated `attendanceGroupKey`
(merge.ts ~L58, derive-dex.ts L71) keys unbound shows by `date:${date}`, collapsing any
two unbound same-date shows — a Phase-5 D-11 multi-device dedup choice that over-collapses
genuine doubleheaders.

Fix (Mechanism A, per 12-RESEARCH.md §SAFE-04): key the UNBOUND branch by
`date:${date}#${sessionId}` so every unbound session becomes its own attendance. The
`id:${showId}` branch is UNTOUCHED, so every show_id join and online multi-device dedup are
preserved exactly. Unify the two duplicated functions into ONE shared pure core module to
kill the drift risk (same class of bug D-07 fixes for `triggerDownload`).

Purpose: the exported backup / dex must never silently lose a caught doubleheader — the core
failure this project exists to prevent (D-01 non-negotiable).
Output: a shared `attendance-key.ts`, both grouping sites rewired onto it, and the two
intentionally-inverted regression tests plus retained bound-dedup tests.
</objective>

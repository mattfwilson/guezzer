<objective>
Fix the two prediction-model correctness bugs in pure core (no app wiring here — that is plan 11-05):
- **PRED-02 (era-prior unit fix):** add `showCount` to `MatrixIndex`, change `eraPrior`'s denominator from the catalog-marginal `basePlayRate` to the dimensionally-correct per-show career rate `node.playCount / index.showCount`, and rescale `eraPriorSmoothingK` from `1` to a per-show value (~0.08). This makes `eraPriorFloor` reachable and flips the 11-01 RED test to GREEN.
- **PRED-01 / PRED-03 (run-grouping):** add a pure `currentRunShowSets` that groups prior finalized shows into the current run by date gap (`runGapDays`, D-02/D-03) and excludes shows on/after a manual reset boundary (D-04). This produces the `recentShowSongSets` window that the already-correct `rotationSuppression` is currently starved of. Core scoring (`rotationSuppression`) is UNCHANGED.

Purpose: `eraPrior` today compares incommensurable units (~100× scale gap) so the retired-song floor never fires (RESEARCH §PRED-02). Separately, `rotationSuppression` is correct but always fed `[]` (the app hardcodes the 3rd `buildShowContext` arg) so cross-night suppression can never fire. This plan fixes the model math and produces the run window; plan 11-05 wires the app query + reset control.
Output: Corrected `eraPrior`, `MatrixIndex.showCount`, `currentRunShowSets`, `runGapDays` + rescaled `eraPriorSmoothingK`, run-grouping tests, barrel exports.

<objective>
Build the identity chrome (AUTH-03/04/07) and the calm reconnect affordance (AUTH-08): a deterministic color+initials avatar in the header (self-sourcing the current identity via `useAuthIdentity`), a bottom sheet showing the full `display_name` + a neutral "Sign out" control, and a new "reconnecting" state on the existing `SyncDot` — never a second connection indicator, never a jarring logout. It depends only on Plan 01 (palette + `identityColorIndex` + copy) and Plan 03 (`useAuthIdentity`), so it runs in Wave 2 parallel to the sign-in surface (Plan 04). The full sign-out teardown (gate re-render) is wired by Plan 06; here the sign-out control clears the record and calls `supabase.auth.signOut()`.

Purpose: A friend must see who they are and be able to hand a shared device to another friend (sign out). The avatar/color is the reusable primitive for Phase 19 friend rows + Phase 20 presence dots. The reconnect affordance keeps the offline→online transition calm at a bad-signal venue.
Output: `IdentityAvatar.tsx` (+ sheet), `AppShell.tsx` header wiring, `SyncDot.tsx` reconnecting state, tests.
</objective>

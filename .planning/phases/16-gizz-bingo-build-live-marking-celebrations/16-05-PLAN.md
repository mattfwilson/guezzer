<objective>
Build the three-tier celebration layer (D-05/D-17/D-18): per-square stamps on every mark, medium badge toasts on four-corners / X / subsequent lines, and a big non-blocking supernova on the first line and blackout ONLY (≤2 big moments/show). It fires app-wide via a module-emitter + App-level host cloned from `BackupToast`, driven by an App-level hook that re-derives the live board and diffs win/mark transitions in a `useRef` so each celebration fires exactly once. Reduced-motion swaps the supernova for a static full-bloom badge crossfade.

Purpose: The reveal is a pillar of the fun — but the live log is sacred (D-17). The supernova bends around the log loop (pointer-events-none, auto-fade), never the reverse.
Output: BingoCelebration (emitter + host + 3 tiers), useBingoCelebrations (transition driver), App-level mount, and the fire-once/budget test.
</objective>

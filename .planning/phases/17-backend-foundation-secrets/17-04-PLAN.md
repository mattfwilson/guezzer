<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| owner machine → dashboard/secrets | Real secrets captured and stored locally |
| CLI → remote DB | `db push` authenticated by DB password + PAT |
| service_role (runtime) → Postgres | Bypasses ALL RLS — must never touch bundle/git |
| authenticated user → other users' rows | RLS write-own enforced live |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-17-09 | Information Disclosure | DB password / PAT / service_role in `.env` | mitigate | `.env`/`.env.local` gitignored (17-02) and shell-exported at run; `.env.example` valueless; `git check-ignore` confirmed in Task 1 |
| T-17-10 | Tampering / Elevation of Privilege | RLS actually enforced post-push | mitigate | Post-push verification: three policies present, RLS enabled, cross-user write rejected, `pg_publication_tables` membership confirmed (Task 2) |
| T-17-11 | Broken Authentication | shared/weak account passwords | mitigate | Distinct per-person passwords required in `.env` (D-06); no self-signup (owner-minted only) |
| T-17-SC | Tampering | service_role misuse | mitigate | service_role used ONLY by the Node seed (`process.env`), never in the Vite bundle; no package installs in this plan |
</threat_model>

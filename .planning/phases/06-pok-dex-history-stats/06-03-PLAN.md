<objective>
Build the pure-core stats engine the entire phase renders: rarity tiers over corpus play data (D-15), the single `deriveDex` derivation entry point (DEX-03/DEX-04, STAT-01/03/04, D-05/D-11/D-12 — derivation is what makes unmark free), and recap assembly (SHOW-14, STAT-02, D-14). All fixture-tested per EVAL-05, which names dex derivation explicitly.

Enabling substrate for the user story: *As a collector, I want every count derived from where I actually was, so that my dex is honest and un-fakeable.* RESEARCH's primary recommendation: one derivation pipeline (`deriveDex`) serves the dex header, album rows, recap "+N new", the compare view (run twice — plan 06-09), and the share card.

Purpose: strict core/UI separation (project constraint) — the app only renders these outputs via useLiveQuery. Storing derived counts is the locked anti-pattern (D-05/D-11/D-12).
Output: three pure modules + shared synthetic fixtures + tests + barrel exports.
</objective>

<objective>
Close the read/export isolation surface of AUTH-05 (D-09/D-11). Plan 02 added the `userId` field + claim and Plan 06 scopes `useDexStats`, but four other namespaced-table consumers still read UNSCOPED — a borrowed-phone signer (identity B) would see identity A's shows, entries, and bingo. This plan scopes every namespaced-table read in `ShowsList`, `ArchiveBrowser`, `RecapView`, and `GamesView` to the current userId (self-sourced via `useAuthIdentity`, mirroring `useDexStats`), and completes the export/import scoping by changing `snapshot()`/`importSnapshot()` to require a `userId` and wiring `readIdentityRecord()?.userId` into their two non-React callers (`exportDownload.ts`, `importPicker.ts`) — all in one plan so `tsc` never sees a signature/caller mismatch. It runs in Wave 2 (needs Plan 02's `userId` field + Plan 03's identity accessors), completing BEFORE Plan 06's Wave-3 device-UAT so the on-device cross-identity isolation check (a second friend sees an empty dex) is actually true across the whole app.

Purpose: The offline-safe identity is worthless if any single unscoped read leaks another friend's dex on a shared phone. This plan owns the remaining leak surface so D-09 holds everywhere, not just in `useDexStats`.
Output: scoped `ShowsList.tsx`/`ArchiveBrowser.tsx`/`RecapView.tsx`/`GamesView.tsx`; scoped `snapshot`/`importSnapshot` + wired `exportDownload.ts`/`importPicker.ts`; `authViewScoping.test.tsx` + an export→import round-trip isolation assertion.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| identity A ↔ identity B on one physical device | Two friends' Dexie rows coexist on a shared/borrowed phone; only the current identity's rows may be read into a view or into an export |
| exported JSON backup → any recipient | A backup file may be handed to a friend; it must carry only the exporter's own identity rows |

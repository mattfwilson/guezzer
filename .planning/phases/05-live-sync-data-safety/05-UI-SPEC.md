# Phase 5 — UI Design Contract

> Visual and interaction contract for Live Sync & Data Safety: the polite `latest` poller's advisory suggestion strip, the quiet sync/offline indicator, and the prominent JSON export/import Settings surface. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **This phase INHERITS the Phase 3 + Phase 4 foundation** (`.planning/phases/03-app-shell-pwa-foundation/03-UI-SPEC.md`, `.planning/phases/04-show-mode/04-UI-SPEC.md`): spacing scale, typography (2 weights / 4 sizes), the dark-only 60/30/10 chrome palette, the tuning-family data palette, the hit/miss semantic pair, the 44px tap floor, `lucide-react` icons, and the system font. **Tokens are NOT re-derived here — they are reused verbatim.** New for Phase 5: the advisory **SuggestionStrip**, the quiet **SyncDot** (online/offline), the **SettingsView** (export/import + storage status), and the End-Show auto-backup nudge. No new colors, no new sizes, no new weights are introduced — a deliberate constraint so the venue UI stays calm and the checker's color/typography dimensions pass by inheritance.

## Spacing Scale

Inherited verbatim from Phase 3 (multiples of 4, Tailwind v4 4px base). **Do not re-declare in code — reuse the existing `@theme` scale.**

| Token | Value | Usage (Phase 5 additions) |
|-------|-------|---------------------------|
| xs | 4px | Sync-dot-to-label gap, suggestion eyebrow-to-name gap |
| sm | 8px | Suggestion-row inter-element gap, settings-list row gap |
| md | 16px | SuggestionStrip padding, SettingsView gutters |
| lg | 24px | SettingsView section vertical rhythm, backup-nudge dialog padding |
| xl | 32px | SettingsView top spacing |
| 2xl | 48px | Major section breaks (SettingsView) |
| 3xl | 64px | Page-level spacing (unused this phase) |

**Exceptions (functional, inherited + extended):**
- **44px minimum interactive target** — hard floor for every tappable element (inherited). In Phase 5 this covers: the suggestion **Adopt** (row) and **Dismiss** (`X`) controls, the fill-`???` control, the **Export backup** / **Import backup** buttons, and the AppMenu **Settings** entry. The SyncDot itself is a passive indicator (not interactive) and is exempt from the tap floor.
- **SuggestionStrip reserved height** — the strip occupies a **fixed-height slot** (`SUGGESTION_STRIP_HEIGHT`, config, default **56px**) in the ShowView column directly above the ActionBar. The slot is **always allocated** so that a suggestion appearing or being dismissed **never re-lays-out the orbit fan** — this preserves SHOW-02's hard "tap targets never move on their own" guarantee. When there is no suggestion, the slot is calm negative space (no persistent empty-state card).
- **Safe-area insets** via `env(safe-area-inset-*)` — inherited; the SettingsView bottom content and any Phase-5 dialog must respect the home-indicator inset.

## Typography

Inherited: **two weights only** (regular 400, semibold 600), **four sizes**. No new sizes or weights. Phase 5 usage mapping:

| Role | Size | Weight | Line Height | Phase 5 usage |
|------|------|--------|-------------|---------------|
| Body | 16px | 400 | 1.5 | Suggestion song name, SettingsView descriptions, storage-status body, import result/error body, offline reassurance text |
| Label | 14px | 600 | 1.4 | SyncDot label (when shown), suggestion eyebrow (`kglw editor` / `Fill in ???`), Export/Import button labels, AppMenu Settings entry |
| Heading | 20px | 600 | 1.2 | SettingsView section headings, backup-nudge dialog heading, import-error dialog heading |
| Display | 28px | 600 | 1.2 | Reserved — not required this phase |

Notes:
- Suggestion eyebrow is **Label** in `text-text-muted`; the suggested **song name is Body** in `text-text-primary`. The provenance framing ("second set of eyes") is carried by the muted eyebrow, never by shouting.
- The **SyncDot's optional label** (e.g. "Offline") is Label-size muted; the offline reassurance sentence is Body-size muted.
- SettingsView import/export descriptions never drop below 16px (readability; the import file-picker text field, if any, must be ≥16px to avoid iOS form-zoom — same rule as the Phase-4 search field).

## Color

Dark theme is **mandatory** (inherited — no light theme, no toggle). Phase 5 introduces **zero new color tokens**. All Phase-5 surfaces reuse the inherited chrome palette; the SyncDot is expressed entirely with existing tokens (see B3).

### A. Chrome palette (60/30/10 — inherited, unchanged)

| Role | Value | Usage (Phase 5) |
|------|-------|-----------------|
| Dominant (60%) | `#0C0C10` | App / orbit-stage / SettingsView background |
| Secondary (30%) | `#17171F` | Elevated surfaces: SuggestionStrip, SettingsView cards/rows, backup-nudge dialog, import-error dialog |
| Accent (10%) | `#F2C14E` (warm gold) | See reserved list |
| Text primary | `#F5F5F7` | Suggestion song name, settings headings, storage-status heading |
| Text muted | `#A1A1AA` | Suggestion eyebrow, offline reassurance, storage-status body, SyncDot (see B3) |
| Border / hairline | `#2A2A34` | SuggestionStrip top edge, settings-row separators, dialog separators |

**Accent gold reserved for (explicit — never "all interactive elements"):**
1. The **Export backup** primary CTA fill in SettingsView — the single most important data-safety tap (the "losing a phone can't lose a dex" action, PWA-04). This is the Phase-5 equivalent of Phase-4's accent-reserved **Start Show**.
2. The **focus-visible ring** on keyboard/AT focus (inherited).

Everything else — **Import backup**, the suggestion **Adopt**/**Dismiss** controls, the fill-`???` control, the AppMenu **Settings** entry — uses text-primary / text-muted on the secondary surface, matching the Phase-4 non-accent control idiom (Search / ??? / Undo). Import is intentionally **not** accent so the one gold CTA (Export) stays unmistakable and the venue's calm is preserved.

### B1. Tuning-family DATA palette (inherited — unchanged)

Colors orbs and the center node (Phase 4). **Not used by any Phase-5 surface** — suggestions are advisory chrome, not orbit data, and are deliberately rendered in neutral text tokens, never in a tuning-family color (a suggestion is not a prediction). Recorded here only to note the intentional non-use.

### B2. Hit / miss semantic pair (inherited — unchanged)

`#22C55E` (hit) / `#EF4444` (miss = Destructive). **Not overloaded by Phase 5.** An adopted suggestion is classified through the *same* Phase-4 hit/miss rule after it is logged (D-03) and wears the standard trail ring at that point — but the SuggestionStrip itself never uses hit-green or miss-red, and the SyncDot never uses them (see B3 rationale).

### B3. SyncDot — expressed with inherited tokens only (D-08, NEW behavior, NO new color)

The sync indicator is a **quiet small dot**, never a loud banner (D-08). It deliberately avoids hit-green and miss-red so it never competes with, or is mistaken for, orbit scoring semantics in the same view. It is expressed entirely with existing tokens by varying **fill vs. outline**, not hue:

| State | Rendering | Meaning |
|-------|-----------|---------|
| Online / polling healthy | **Filled** dot, `text-muted` `#A1A1AA` | The poller is active (or last poll succeeded); calm, present, unremarkable |
| Offline / paused | **Hollow** (1px `#A1A1AA` ring, transparent fill) dot | `navigator.onLine === false`, or backgrounded/paused; the dot dims to a ring — plus a one-time calm reassurance line (copy below) |

Rationale (checker color dimension): no new token, accent gold untouched, and the hit/miss reds/greens are not overloaded. The `#A1A1AA`-on-`#17171F` dot clears the ≥3:1 graphical-contrast floor (WCAG 1.4.11). The dot's diameter is **8px** (`sm`-adjacent, a deliberate fixed exception — it is a status glyph, not a control), placed in the Show header row next to the tally/date stamp.

**No weak-fan-style softening, no toast, no color flashing** — per D-08 the transition online↔offline is expressed only by fill↔outline plus the one-time text; polling resumes silently.

### Destructive actions

**None this phase — by design.** Phase 5 ships **no destructive UI**:
- **Import** is a stable-key **union merge that never drops local data** (D-10) — non-destructive by construction; it needs no confirm dialog.
- **Dismissing a suggestion** only hides an advisory row — the editor never overwrites manual tracking (SYNC-02); nothing is deleted.
- **Auto-bind** (D-07) only *adds* `show_id`/venue to a provisional record and is reversible via export/import; it is silent and non-destructive.
- The destructive **"clear all data" / reset is explicitly deferred** (out of scope — data-loss footgun on a hard-bar phase). The SettingsView MUST NOT render any reset/clear control.

Consequently there is no destructive-confirmation row to specify. Any future destructive control would inherit the Phase-4 destructive pattern (`#EF4444`, explicit confirm dialog) — but none exists in Phase 5.

## Layout & Interaction Contract

### Show view additions (within `#/show`)

Region stack, extending the Phase-4 ShowView flex column (top → bottom):

1. **Header row (extends Phase-4):** unchanged, plus the **SyncDot** (§Color B3) placed adjacent to the existing tally/date stamp, top-right. Filled = online, hollow = offline. Passive; not a tap target.
2. **Comet-trail strip** — unchanged (Phase 4).
3. **Orbit stage** — unchanged (Phase 4). Its container height is **stable** because the SuggestionStrip slot below is fixed-height (see 4) — orbs never re-lay-out due to a suggestion.
4. **SuggestionStrip (NEW, D-01):** a **fixed-height (56px) slot** on the secondary surface, directly above the ActionBar, with a `#2A2A34` top hairline. Shows the **next 1–2 un-logged editor songs** (`SUGGESTION_COUNT`, config, default 2), deduped by song ID against the current trail (D-02), plus any **fill-`???`** hints (D-04). Each row: muted eyebrow + song name + a `Plus`/**Add** adopt control and an `X` dismiss control; the row also supports **swipe-to-dismiss**. It is **below the orbit, never inside it** (mis-tap avoidance in the dark, contra ghost-orbs, D-01). When empty, the slot is blank calm space.
5. **Bottom action bar** — unchanged (Phase 4).

### Settings surface (NEW route)

- **Placement (D-14, Claude's-discretion decision):** add `"settings"` to the `ROUTES` allow-list in `useHashRoute.ts` (a validated security allow-list — extension is the correct pattern) and reach `#/settings` via a **Settings entry in the existing AppMenu** (opened by the already-present header `Menu` button). **Do NOT add a 4th bottom tab** — the 3-tab bar (Show/Explore/Dex) and the one-thumb reach zone stay uncrowded, and the existing header menu is always reachable, satisfying "prominent, always available" (D-13) without new navigation chrome. The `AppShell` header `Menu` button (currently wired to `onMenuClick`) opens the AppMenu; the new **Settings** row navigates to `#/settings`.
- **SettingsView content (scope-limited, D-14):** a scrolling view on the dominant surface with one **Backup & data** section containing: the **Export backup** button (accent), the **Import backup** button (neutral), and the **storage-protection status** readout (`ShieldCheck`/`ShieldAlert` + copy). **No destructive controls** (no clear/reset — deferred).

**Interaction rules:**
- **Tap Add (adopt suggestion)** → runs the **same Phase-4 hit/miss classification** and logs the song with `source: 'editor'` provenance (D-03); the trail/tally re-render via `useLiveQuery`. The adopted row leaves the strip (now logged → deduped out, D-02). No confirm (advisory-to-logged is a fast path, mirroring orb-tap).
- **Tap `X` / swipe** to **dismiss a suggestion** → the row is removed from the strip; non-destructive (nothing logged, manual tracking untouched, SYNC-02). A dismissed song may reappear only if it remains the next un-logged editor song and the strip re-derives — acceptable; keep it simple (no persistent per-song "muted" state required this phase).
- **Tap fill-`???`** → opens the Phase-4 rename path pre-filled with the editor's song name (D-04/D-14/D-15); user confirms or edits. **Never auto-applied.**
- **Tap Export backup** (Settings) → serializes to JSON and triggers an anchor download; confirmation "Backup saved."
- **End Show** → auto-downloads the JSON (D-13) with the muted "Backup saved to your downloads." confirmation. Not a per-show nag beyond this one confirmation.
- **Tap Import backup** → opens the file picker (`<input type="file" accept="application/json">`); on a valid file, union-merge and show "Backup merged." with counts; on a corrupt/unrecognized file, **reject with the error dialog and change nothing** (D-12).
- **Offline (SYNC-03):** the SyncDot flips filled→hollow and the one-time reassurance line shows; the app stays fully functional; polling resumes silently on reconnect — **no banner, no blocking state** (D-08).
- **Contrast/a11y floor (inherited):** all text ≥ 4.5:1 on its surface; the SyncDot ≥ 3:1 graphical; focus-visible ring on all interactive Phase-5 controls; the passive SyncDot carries an `aria-label` reflecting online/offline for AT (e.g. "Sync: offline").
- **Untrusted content (SECURITY):** editor-sourced strings from `latest` (`songname`, `venuename`) and all imported-file strings render as **plain escaped text only — never `dangerouslySetInnerHTML`** (React default escaping is sufficient).
- **Motion:** the SuggestionStrip row may fade/slide within its fixed slot (respect `prefers-reduced-motion`); the slot height never animates (that would move orbs). The SyncDot fill↔outline change is instant or a gentle opacity crossfade, never a color pulse.

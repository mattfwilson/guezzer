<objective>
Build the pure, zero-DOM export/import core for Phase 5 data safety: a versioned export envelope schema, a pure `serializeExport`, and a `parseAndMergeImport` that validates, version-migrates, union-merges (never dropping local data), and same-show-dedupes — all Node-testable. The app tier (Plan 05) only does the DOM download/upload and the Dexie write.

This plan is the enabling substrate for the data-safety vertical slice whose user story is: *As a dex owner, I want to export and import all my data as a JSON file, so that losing my phone never means losing my dex (PWA-04).*

Purpose: PWA-04 — the JSON export is the genuine eviction backstop (iOS Safari can evict IndexedDB). The merge must be friend-safe by construction (stable-key union) and refuse to partial-merge a bad file (D-10/D-11/D-12).
Output: `data-safety/export-schema.ts`, `serialize.ts`, `merge.ts`, barrel exports, and two Node test files.
</objective>

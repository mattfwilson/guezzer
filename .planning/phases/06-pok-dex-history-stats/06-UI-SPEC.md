# Phase 6 — UI Design Contract

> Visual and interaction contract for Pokédex, History & Stats: the album-shelf dex, retroactive attendance marking, the post-show recap with game-style rarity, show history, friend-file compare + PNG share card, and the folded Show-Mode polish pass (FAB menu, orb-label fit, InstallBanner frequency). Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **This phase INHERITS the Phase 3 + 4 + 5 foundation** (`03-UI-SPEC.md`, `04-UI-SPEC.md`, `05-UI-SPEC.md`): spacing scale, typography (2 weights / 4 sizes), the dark-only 60/30/10 chrome palette, the tuning-family data palette, the hit/miss semantic pair, the 44px tap floor, `lucide-react` icons, and the system font. **Tokens are NOT re-derived here — they are reused verbatim.** New for Phase 6: the **rarity-tier data ramp** (the phase's one new color axis, D-15), the **caught/unseen dex semantics** (D-05/D-06), album-cover thumbnails as a bundled asset class (D-03), the **FAB action menu** replacing the ActionBar (D-20), and the recap/compare/share-card surfaces.
>
> **Inherited-state amendment (recorded, not re-litigated):** quick task `260713-w1c` (2026-07-14, owner override of 05-UI-SPEC §B3) changed the SyncDot online state from muted-filled to **hit-green `#22C55E` filled**. Hit-green therefore now carries a general "positive / healthy / yes" meaning beyond trail rings — Phase 6's caught-checkmark usage (below) extends that owner-set precedent deliberately.

## Spacing Scale

Inherited verbatim from Phase 3 (multiples of 4, Tailwind v4 4px base). **Do not re-declare in code — reuse the existing `@theme` scale.**

| Token | Value | Usage (Phase 6 additions) |
|-------|-------|---------------------------|
| xs | 4px | Rarity-badge internal padding, check-to-name gap, cover-to-tally gap |
| sm | 8px | Album-grid gutter, song-row inter-element gap, tier-chip gaps in recap |
| md | 16px | Dex screen gutters, album-card padding, recap section gaps, archive row padding |
| lg | 24px | Recap section vertical rhythm, compare-view column gap, dialog padding |
| xl | 32px | Dex header block spacing, recap hero spacing |
| 2xl | 48px | Major section breaks (recap setlist → CTAs) |
| 3xl | 64px | Dex empty-state top spacing |

**Exceptions (functional, inherited + extended):**

- **44px minimum interactive target** (inherited hard floor) — covers: every album card, every song row (rows are informational but tappable where they open detail), every archive show row, mark/unmark controls, segment-control halves, year-section headers, FAB main button and **every expanded FAB action row**, recap CTAs, compare-view close, share/download buttons.
- **FAB main button = 56px diameter** (`ui.FAB_DIAMETER`, config) — matches the Phase-4 orb visual minimum; the venue's most-tapped control must be findable by feel. Expanded action rows are ≥ 48px tall (`ui.FAB_ACTION_HEIGHT`), full hit-area including their labels.
- **FAB anchor offset** — `position: fixed`, bottom-right; bottom offset = `env(safe-area-inset-bottom)` + BottomTabBar height + SuggestionStrip slot (56px) + `sm` gap, right offset = `md` + `env(safe-area-inset-right)`. Coordinate with `pwa/bottomOverlayInset.ts`. The FAB must never overlap the SuggestionStrip's dismiss (`X`) control.
- **Album cover thumbnails = 80px square displayed** (`dex.ALBUM_ART_DISPLAY_PX`), shipped as 160px @2x assets (see §Assets). 80 is a 4-multiple; the album card's total tap area is the whole card (≥ 44px trivially).
- **Orb-label fit floor (D-21)** — see Typography exception below; a text-fitting rule, not a spacing token.
- **Safe-area insets** via `env(safe-area-inset-*)` — inherited; the FAB, recap footer CTAs, and archive list bottom must respect the home-indicator inset.

## Typography

Inherited: **two weights only** (regular 400, semibold 600), **four sizes**. No new sizes or weights. Phase 6 usage mapping:

| Role | Size | Weight | Line Height | Phase 6 usage |
|------|------|--------|-------------|---------------|
| Body | 16px | 400 | 1.5 | Song-row names, archive venue/city lines, recap body copy, compare-view diff rows, empty-state bodies |
| Label | 14px | 600 | 1.4 | Album-card tallies (`tabular-nums`), rarity/debut badges, segment-control labels, stat sublines (sightings · gap · last seen), FAB action labels, year headers |
| Heading | 20px | 600 | 1.2 | Album names (card + detail header), recap section headings, "Rarest catch" name, dialog headings, archive view title |
| Display | 28px | 600 | 1.2 | **The dex completion % headline** (`87/264 · 33%`, `tabular-nums`) and the recap hero tally — the collection's single biggest number moments. First real use of Display since Phase 3 declared it. |

Notes:

- All counts, percentages, dates, and gaps render with `tabular-nums` (inherited idiom) so lists don't shimmer as data changes.
- Rarity/debut badges are Label-size text in a hairline-bordered pill — **the tier word always renders; color is reinforcement only** (color-blind safety, WCAG 1.4.1).
- **Orb-label fit exception (D-21, documented functional exception):** inside prediction orbs and the center node, song names use **scale-to-fit**: start at the role size (Label 14px for orbs, Heading 20px for the center node), wrap to at most `show.ORB_LABEL_MAX_LINES` (default 2; center node 3), and shrink the font uniformly to a hard floor of `show.ORB_LABEL_MIN_FONT_PX` (default **11px**; center node floor 14px) before ellipsizing. This mirrors the 44px-floor style of exception: a geometry-constrained fitting rule inside a fixed circle, not a new type role. Weight stays 600; the full name remains available via the `Info` "why" detail. Constants live in `config.ts` (single-config rule).
- Body never below 16px in free-flowing text (inherited); the archive search field input is ≥ 16px (iOS form-zoom avoidance, same rule as Phase-4 search).

## Color

Dark theme is **mandatory** (inherited — no light theme, no toggle). Phase 6 introduces **one new color axis** — the rarity-tier ramp (D-15) — adding **two new hues** (`#60A5FA`, `#E879F9`). Everything else reuses inherited tokens.

### A. Chrome palette (60/30/10 — inherited, unchanged)

| Role | Value | Usage (Phase 6) |
|------|-------|-----------------|
| Dominant (60%) | `#0C0C10` | Dex / recap / archive / compare backgrounds; share-card background |
| Secondary (30%) | `#17171F` | Album cards, song-row list surface, archive rows, recap cards, compare panels, FAB button + menu, dialogs |
| Accent (10%) | `#F2C14E` (warm gold) | See reserved list |
| Text primary | `#F5F5F7` | Album names, song names, recap numbers, stat values |
| Text muted | `#A1A1AA` | Stat sublines, year headers, unseen silhouettes, empty-state bodies, Common tier |
| Border / hairline | `#2A2A34` | Card edges, row separators, badge outlines, segment-control track, FAB border |

**Accent gold reserved for (explicit — never "all interactive elements"):**

1. The **Share card** primary CTA (dex header + recap footer) — the phase's payoff action (SHAR-02).
2. The **Legendary rarity tier** (badge text/border + share-card tier count) — deliberate, documented exception to "accent is chrome-only": Legendary is scarce *by construction* (bottom ~5% play-rate quantile, D-15), so gold stays rare on screen, and "legendary = gold" is exactly the game-language the owner asked for.
3. The **focus-visible ring** on keyboard/AT focus (inherited).
4. The **active segment** indicator in the Albums | Shows segment control (one half at a time — the dex's equivalent of the inherited active-nav-tab accent use).

Everything else — Mark attended controls, Import/compare controls, FAB (all states), year headers, album cards — uses text-primary/text-muted on the secondary surface. **The FAB is deliberately NOT accent**: it is the venue's most-tapped chrome and must not compete with Start Show or orb data; a 56px elevated circle with a hairline border is findable in the dark without gold.

### B1. Tuning-family DATA palette (inherited — unchanged)

`standard #5EC8E5` / `C# standard #B98CF2` / `microtonal #FF8A5B`. Used only by orbs/center node (Phase 4) and later the constellation (Phase 7). **Phase 6 surfaces do not use tuning colors** — dex rows, recap setlists, and badges stay in neutral + rarity tokens so one row never carries two competing color meanings. (The recap setlist's hit/miss rings are the B2 pair, not tuning.)

### B2. Hit / miss semantic pair (inherited + one extension)

`#22C55E` (hit) / `#EF4444` (miss = Destructive). Phase 6 usage:

- **Recap setlist rings:** each recap setlist row wears its Phase-4 hit/miss ring verbatim — same meaning, same colors.
- **Caught state (NEW, sanctioned extension):** the dex's caught tick (`Check` in song rows, `CircleCheck` on marked archive shows, fully-completed album tick) renders in **hit-green `#22C55E`**. Rationale: the owner's `260713-w1c` override already widened hit-green to a general "positive/healthy" signal (SyncDot online), and "caught" is the dex's success semantic. No new green is introduced; do not add one.
- **Destructive:** `#EF4444` for the **Unmark show** confirm action (the phase's only destructive control). Same red as miss rings by inherited design; do not add a third red.

### B3. Rarity-tier DATA ramp (NEW this phase — D-15)

Colors the Common/Uncommon/Rare/Legendary badges in dex song rows, the recap tier breakdown, "rarest catch" highlights, and the share card. Thresholds are **data-driven quantiles over corpus play rate** (constants in config — see §Config); the ramp below is the visual contract.

| Tier | Value | Token status | Notes |
|------|-------|--------------|-------|
| Common | `#A1A1AA` | reuses text-muted | Unremarkable by design — most of the catalog |
| Uncommon | `#60A5FA` (blue) | **NEW** | Blue is a hue currently unused anywhere in the app — zero collision |
| Rare | `#E879F9` (fuchsia) | **NEW** | Deliberately hotter/pinker than tuning C#-violet `#B98CF2`; the two never co-occur in Phase 6 surfaces (B1 non-use) |
| Legendary | `#F2C14E` (gold) | reuses accent | The documented exception in the accent reserved list above |

Rendering rules (checker-verifiable):

- A tier badge is a Label-size pill: tier word in the tier color, 1px border in the tier color at ~40% opacity, transparent fill on the secondary surface. **Never color-only** — the word always renders (color-blind safety).
- All four tier colors clear **≥ 3:1 graphical contrast** against `#17171F` and `#0C0C10` (WCAG 1.4.11); as badge *text* they pair with the border/pill so the word also clears ≥ 4.5:1 — `#60A5FA`, `#E879F9`, `#F2C14E`, and `#A1A1AA` all pass on `#17171F`.
- The ramp is **data semantics, not chrome**: never use tier colors for buttons, surfaces, or focus states.

### B4. Unseen / silhouette treatment (D-06)

Never-seen songs and uncaught albums render **dimmed in the same view** (no separate "missing" list styling): text at `text-muted`, leading tick slot replaced by a hollow `#2A2A34`-bordered circle, album covers at **40% opacity + grayscale filter** when an album has zero catches. This is the same visual language Phase 7's constellation dimmed-silhouettes will inherit. Dimmed content is still readable (muted-on-elevated ≥ 4.5:1) — dimming signals state, never hides information.

## Assets — Album Cover Thumbnails (D-03, Claude's-discretion spec)

Build-time fetched, committed, bundled — the corpus/matrix pattern. **No runtime cover fetching, ever** (offline-first).

| Property | Contract |
|----------|----------|
| Source | **kglw.net album pages first** (covers are already curated per-album there; reuse the existing paced-fetch + User-Agent etiquette from `fetch-corpus.ts`), falling back to **MusicBrainz Cover Art Archive** for any album kglw.net lacks. One-time script, manual re-run only — never CI-scheduled. |
| Format / size | WebP, **160×160** (2× of the 80px display size), quality ~70, target **≤ 10 KB each** |
| Location | `packages/app/src/assets/covers/{album-slug}.webp`, committed; imported statically so Vite fingerprints + the SW precaches them |
| Budget | ~25–30 grid albums → **≤ ~300 KB total**; the build script fails loudly if an image exceeds 25 KB (guard against accidental full-res commits) |
| Fallback card | Albums with no art, plus the **Miscellaneous** and **Covers** buckets (D-04), render a generated placeholder: album initials (Heading, text-muted) centered on the elevated surface with a hairline border — same 80px footprint, no layout shift |
| Attribution | Source URL per cover recorded in a committed manifest (`covers-manifest.json`) written by the fetch script — personal-use tool, but provenance stays inspectable |

**Grid membership note (UI-consequential):** `data/raw/albums.json` contains **147 raw KGLW "albums"** including singles, promos, and music-video pseudo-albums. The album grid must render the **primary studio discography (~25–30 LP cards) + Miscellaneous + Covers** — never 147 cards. The canonical-album derivation (D-04: `islive=0`, earliest release) must therefore exclude single/promo/video pseudo-albums from *card* status (fold their songs into the LP that carries them, else Miscellaneous). Exact derivation is core-side planner territory; the UI contract is: **the grid reads as a discography shelf, not a database dump.**

## Layout & Interaction Contract

### 1. Dex tab (`#/dex` — replaces `PlaceholderView`)

Vertical stack on the dominant surface:

1. **Dex header:** completion headline (`87/264 · 33%` at Display, `tabular-nums`), sublines for rarest catch (tier badge) + show count, and the accent **Share card** CTA. Generous `xl` rhythm — this is the collection's face.
2. **Segment control:** **Albums | Shows** — a two-half pill on the elevated surface, active half accent-tinted (reserved-use #4), each half ≥ 44px. Segment state is component state within `#/dex` (no new hash routes; the 3-tab bar and `ROUTES` allow-list stay untouched — internal view-state navigation, matching the no-routing-library constraint).
3. **Albums segment (default):** a **2-column card grid** (`sm` gutters, `md` screen gutters) sorted alphabetically by album name (D-02); **Miscellaneous** and **Covers** buckets pinned last. Each card: 80px cover (or fallback initials card), album name (Heading, 2-line clamp), `{caught}/{total}` tally (Label, `tabular-nums`); zero-catch albums dim per §B4. Tap → **Album detail**.
4. **Album detail (drill-in, slides over within `#/dex`):** header (cover + album name + tally + back), then the track-ordered song list. Each row ≥ 44px: caught tick (green `Check`) or hollow unseen circle · song name (Body; dimmed when unseen) · stat subline (Label, muted) · rarity or debut badge trailing. Seen/unseen is **display-only derived state — no toggle affordance exists anywhere** (D-05).
5. **Shows segment:** header row with the **Mark attended shows** CTA (neutral, `Plus`-adjacent styling), then attended shows **newest-first** (D-16): tracked and retro-marked shows in one list, each row `{date} · {venue}`, song count, tally chip when tracked. Tap → recap (tracked) or plain setlist view (retro-marked) with set structure (HIST-01).

### 2. Archive browser — Mark attended (D-09..D-12)

Full-screen view reached from the Shows segment CTA:

- **Search field** at top (≥ 16px input), fuzzy over date/venue/city of the bundled 738-show corpus — corpus-first, fully offline (D-09). Wrap the search as a pure core function (the `searchCatalog` idiom).
- **Year sections, newest first** (D-10), collapsible headers (Label, muted, `tabular-nums` year + show count). Current/most-recent year expanded by default.
- **Show rows** ≥ 44px: date (Label) + venue, city (Body) + trailing state: unmarked (hollow circle) / marked (`CircleCheck` green). **Tap an unmarked row → marks instantly** (one tap, full-setlist credit, D-11) with the "+{n} songs caught" inline flash; counts everywhere recompute via `useLiveQuery`. **Tap a marked row's state control → Unmark confirm** (destructive contract above).
- **Online fallback** (D-09): when online, a **"Search kglw.net for newer shows"** row appears below corpus results (or when a dated query lands past the bundled corpus). Runs the polite live search (core ingestion schemas + `assertFilterApplied`; API etiquette inherited). Offline: the row is hidden, replaced by the muted "Newer shows need a connection." note. Results merge into the same row UI, keyed by stable `show_id` (DEX-02).

### 3. Post-show recap (SHOW-14, STAT-02, D-13/D-14/D-15)

- **Auto-appears immediately after End Show finalizes** (hooks the existing `EndShowDialog → endShow` seam, after the Phase-5 auto-backup) as a full-screen view on the dominant surface — the payoff moment. Also reachable forever from the Shows segment (D-13).
- Section order (top→bottom): heading + date/venue → **hero tally** (Display) → source split line → **rarity block** (score, tier chips ×count, rarest catch of the night) → **+N new catches** (`Sparkles`; omitted at zero) → **final setlist** grouped by set/encore with each row wearing its Phase-4 hit/miss ring + rarity badge → footer CTAs **Share card** (accent) / **Done**.
- Scrollable; footer CTAs respect the safe-area inset. Any bonus recap metrics (D-15 discretion) join the rarity block as additional muted stat lines — honest, corpus-derived, never a new color.

### 4. Friend compare (SHAR-01, D-17)

- **Entry:** Settings → Import backup. After Phase-5 zod validation, fork on the envelope's **owner name/id**: mine → existing merge path verbatim; someone else's → **compare view, zero DB writes**. Files pre-dating the owner field prompt "Whose dex is this?" before opening compare.
- **Layout:** persistent read-only banner (top, muted) → **You vs {name}** stat columns (completion %, songs caught, shows, per-tier counts — `tabular-nums`, side-by-side on the elevated surface) → diff sections **"Only {name} has caught (n)"** / **"Only you have caught (n)"** as collapsible song lists (rarity badges included — trophy browsing) → **Close**. Read-only throughout: no adopt/merge/copy affordance exists (a friend's file can never inflate your attendance).

### 5. Share card (SHAR-02, D-18/D-19)

- **Canvas-rendered PNG, 1080×1350 (4:5)** on `#0C0C10`, drawn from the same derived dex stats (pure core input → canvas draw fn): Guezzer wordmark, completion % (hero), songs caught `{x}/{y}`, show count, rarest catch + tier, **tier breakdown** (`3 Legendary · 12 Rare · …` — Legendary in gold), latest/best show. System-font stack; colors from this contract only.
- **Share:** `navigator.share({ files })` where supported (feature-detect + try/catch, the `persist.ts`/`wakeLock.ts` never-throw idiom); fallback = anchor download + "Card saved to your downloads." Render failure = non-blocking error copy above; the app state is never touched.

### 6. Folded Show-Mode polish (D-20/D-21/D-22)

- **FAB menu (D-20):** `ActionBar` is removed from the ShowView flex column; the orbit stage absorbs the reclaimed height (re-verify the non-scrolling AppShell seam, `scroll=false`, and that the SuggestionStrip slot still sits directly above the BottomTabBar). The FAB (56px, elevated + hairline, `Plus` glyph) is `position: fixed` bottom-right per the §Spacing anchor contract. **Expanded:** a vertical speed-dial opening upward — five rows (icon + Label), each ≥ 48px, hottest nearest the thumb: **Undo, ???, Search, Set break, Encore** (bottom→top); a **scrim** (`#0C0C10` at ~60% opacity) covers the stage so an expanded-menu tap can never hit an orb; tapping the scrim or the FAB (glyph rotated 45° to ✕) collapses. Any action tap **auto-collapses** and then runs the inherited Phase-4 behavior verbatim (???/orb/Undo speeds: collapsed-state costs exactly one extra tap — owner-accepted). The FAB and menu carry the gesture-suppression CSS scope (`touch-action: manipulation`, no select/callout) and respect `prefers-reduced-motion` (expansion becomes instant opacity).

<objective>
SAFE-01 + SAFE-03: the End-Show auto-backup must be honest. Two halves of one principle —
the backup must record a show that is actually finalized (SAFE-01), and the "Backup saved"
signal must reflect a backup that actually happened (SAFE-03).

Root causes (12-RESEARCH.md): `EndShowDialog.handleConfirm` (L82-87) fires
`void endShow(); void exportBackup();` with neither awaited, so the snapshot can capture the
show as still-active (SAFE-01); and a static `CircleCheck` "backup saved" line (L108-112)
renders unconditionally while the dialog is open (SAFE-03).

Fix (D-04/D-05): make `handleConfirm` async — `await endShow(sessionId)` (commit finalize)
→ `onEnded?.` → `onClose()` → `await exportBackup()` → fire an app-level toast ONLY on
`{ ok: true }`. Remove the static markup. The toast MUST be hosted at App level (a new
`<BackupToast/>` sibling of `<UpdateToast/>`) because confirming End Show makes `ShowView`
early-return `<RecapView>`, unmounting the dialog's whole subtree — a dialog- or
ShowView-owned toast would never render.

Purpose: the iOS-eviction backstop backup must not lie about what it contains or whether it
succeeded.
Output: a persistent app-level toast + emitter, the rewired async confirm sequence, and the
extended ordering/confirmation component test.
</objective>

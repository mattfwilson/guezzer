# Quick Task 260716-vw2 — Fix WARNING-1 (own-backup restore fork)

## Problem

`classifyImport()` (`packages/app/src/settings/importPicker.ts`) returned
`kind: "friend"` whenever a backup file carried an `owner` that did not
case-insensitively match the **local** owner name. On a genuine iOS-eviction
reinstall — exactly the scenario PWA-04's export exists for — the local
`ownerName` meta row is evicted along with everything else (→ `null`), so
importing your **own** named backup classified as a friend's file and opened the
read-only CompareView (zero DB writes) instead of restoring. Non-destructive and
recoverable, but it silently routed the user away from the headline phone-loss
recovery flow.

Surfaced as **WARNING-1** by the gsd-integration-checker during the v1.0
milestone audit.

## Fix

When the file **has** an owner but the **local** owner is null/blank (we cannot
determine whose dex it is), route to `kind: "unowned"` — SettingsView's existing
"Whose dex is this?" prompt already handles that kind and exposes an
"It's mine, restore it" button (`confirmPromptMine` → `mergeFile`). Only route to
`"friend"` when a local owner name IS set and differs. Behavior when the local
owner is set is unchanged (mine/friend as before). No UI change required.

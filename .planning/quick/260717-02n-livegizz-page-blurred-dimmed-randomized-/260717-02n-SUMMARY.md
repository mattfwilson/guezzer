# Summary — Quick Task 260717-02n

Replaced the LiveGizz (Show) page's flat background with a randomized bundled
album cover, heavily blurred and dark-dimmed, keeping all body content legible.
Implements todo `2026-07-17-blur-a-random-album-cover-as-the-show-screen-background`.

## Changed

- **`packages/app/src/dex/covers.ts`** — new `coverUrlList()` export (all bundled
  cover URLs).
- **`packages/app/src/config.ts`** — new `config.show.background` tunables:
  `BLUR_PX: 40`, `DIM_OPACITY: 0.72` (single-config rule).
- **`packages/app/src/show/ShowBackground.tsx`** (new) — decorative
  `absolute inset-0`, `aria-hidden`, `pointer-events-none` layer: blurred
  `object-cover` `<img alt="">` + `rgba(12,12,16,DIM_OPACITY)` scrim. Returns
  `null` when no cover → plain surface fallback.
- **`packages/app/src/show/ShowView.tsx`** — picks one random cover ONCE via a
  `useState` initializer in the always-mounted parent (stable across pre-show →
  active → end, no mid-show flicker); a `withBackground()` helper wraps the three
  in-page states (PreShowLauncher, matrix-fail, orbit) with the backdrop behind a
  `z-10` content column. RecapView early-return left unchanged (excluded).
- **`packages/app/src/show/OrbitStage.tsx`** — removed the opaque `bg-surface`
  from the stage root (deviation from plan, which listed only ShowView). Without
  this the in-show orbit stayed flat black and the cover only showed pre-show;
  now the shared dimmed backdrop shows behind the orbs too. Safe: OrbitStage is
  used only inside ShowView and the null-cover fallback is the same color.

## Verification

- `npx tsc -p packages/app/tsconfig.json --noEmit` → exit 0.
- `npx vitest run` → **481 passed / 66 files**, no failures.
- Ran the app (dev server + Playwright, iPhone viewport):
  - Pre-show: blurred warm cover behind "Ready when you are" + gold Start Show
    button — all legible; header/tab bar/install banner stay opaque.
  - In-show orbit: dimmed colorful cover behind "Tap the opener" center node —
    legible; date/tally header, End Show, FAB unaffected; orbit still

  - Exactly one decorative `<img alt="">` with `filter: blur(40px)`; **zero

## Decisions / notes

- **Fully random** cover (not biased to caught songs) — v1 simplicity.
- **Stable per mount** (one cover for the app session on the screen). Possible
  later enhancement: re-pick per show night (key off `activeSessionId`).
- **Offline**: 160×160 WebP thumbs are bundled via `import.meta.glob` + SW
  precached; heavy blur hides the low resolution.
- Blur/dim are config-tunable if the ambient effect needs softening/strengthening.

## Out of scope

RecapView background, caught-bias selection, per-night re-pick, GizzVerse
constellation background. Routes/storage untouched.

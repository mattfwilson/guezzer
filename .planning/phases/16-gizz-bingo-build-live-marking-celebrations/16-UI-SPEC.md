# Phase 16 — UI Design Contract

> Gizz Bingo: Build, Live Marking & Celebrations. Visual and interaction contract for the
> playable/fun surface layered on the finished bingo engine (Phase 14) + persistence/lock/replay
> machinery (Phase 15). Generated by gsd-ui-researcher; verified by gsd-ui-checker.
>
> **Source discipline:** every token below is either (a) DETECTED from the shipped design system
> (`packages/app/src/styles.css` `@theme`, `config.ts`) or (b) DERIVED from the 24 locked decisions
> in `16-CONTEXT.md` (D-01..D-24). No new visual language is introduced — this phase reuses the
> dark galaxy aesthetic, the honest-number precedent, and the rarity/caught color language already
> in the app. Nothing was re-asked of the user; all contract fields were pre-populated.

## Spacing Scale

Tailwind v4 default 4px scale (detected in use across `GamesView`, `RecapView`, `ShowView`).
All multiples of 4.

| Token | Value | Usage (this phase) |
|-------|-------|--------------------|
| xs | 4px | Board square inner gap, stamp/glyph inline gap, chip padding |
| sm | 8px | Grid gap between the 4×4 squares (`gap-2`), meter bar → caption gap, toast icon → text gap |
| md | 16px | Screen horizontal padding (`px-4`), sheet section padding, deal-button internal padding |
| lg | 24px | Vertical rhythm between deal screen blocks (`gap-6`), meter block → board gap |
| xl | 32px | Deal-screen top padding (`pt-8`), section breaks |
| 3xl | 64px | Scroll bottom padding above the tab bar (`pb-16`) |

Exceptions:
- **44px minimum tap target** (`min-h-11` / `min-w-11`) on every interactive element — deal
  buttons, swap-sheet rows, each of the 16 board squares, the peek-strip tap zone, the "Not now"
  nudge dismiss. Venue reality: fat-thumb, in the dark (matches the app-wide `min-h-11` precedent).
- **Safe-area insets** honored via `env(safe-area-inset-*)`; top-anchored surfaces use
  `calc(env(safe-area-inset-top) + 12px)` — NEVER a bare inset (the doubled-inset bug). GamesView
  content renders below the header that owns the notch inset, so it uses a plain `pt-8` (matches
  the shipped GamesView / SettingsView precedent — do not re-apply the top inset there).
- **Board square minimum**: each of the 16 squares must render ≥ 44×44px tappable even on the
  narrowest target (iPhone SE, 320px viewport → after 16px side padding and 3× 8px gaps, a 4-col
  grid yields ~65px squares; floor holds).

## Typography

Detected sizes/weights from shipped components (`GamesView`, `RecapView`, `DexHeader`). Exactly
**4 roles, 2 weights**. Weights: regular **400** + semibold **600** (no other weight in the app).

| Role | Size | Weight | Line Height | Usage (this phase) |
|------|------|--------|-------------|--------------------|
| Label | 14px (`text-[14px]`) | 400 / 600 | 1.25 (`leading-tight`) | Fire-rate % chips, meter caption, swap-sheet section headers, "Locked at Start Show" explainer, gamble hints under vibe buttons |
| Body | 16px (`text-base`) | 400 | 1.5 (`leading-normal`) | Deal-screen body copy, one-away banner text, mark-toast caption, empty-state body, nudge body |
| Heading | 20px (`text-[20px]`) | 600 | 1.2 (`leading-tight`) | Deal-screen title, vibe button labels, badge-toast label, sheet title, share-entry heading |
| Display | 28px (`text-[28px]`) | 600 | 1.2 | Supernova headline ("First line!" / "BLACKOUT!"), expected-marks figure "~11/15" emphasis; `tabular-nums` on all numeric figures |

Notes:
- **Numeric figures use `tabular-nums`** (fill "~11/15", fire-rate "89%", win badges) — matches the
  recap hero-tally / orb-percent precedent so digits don't jitter.
- App-wide display exception exists (56px doubled dex headline, `DexHeader`); **not used in this
  phase** — 28px is this phase's display ceiling.

## Color

Detected `@theme` tokens (`styles.css`). The 60/30/10 split is already the app's contract; this
phase does **not** introduce a new surface or accent hue.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0c0c10` (`--color-surface`) | App/screen background; the galaxy backdrop (reused from `ConstellationCanvas`/`ExploreBackground`) sits behind the deal screen, board, and supernova |
| Secondary (30%) | `#17171f` (`--color-elevated`) | Cards, the swap bottom-sheet, the 4×4 board frame, unmarked squares, the LiveGizz peek strip, mark/badge toasts, the fill-meter track |
| Accent (10%) | `#f2c14e` (`--color-accent`) | **Reserved — see list below** |
| Destructive | `#ef4444` (`--color-destructive`) | The reshuffle "discard custom swaps?" confirm action only (D-06) |

**Accent (`#f2c14e`) reserved for — explicit list (never "all interactive elements"):**
1. Focus rings (app-wide a11y, unchanged) — including on the D-24 focusable square toggle buttons.
2. The **one-away target-square glow/pulse ring** (D-13) — the single closest near-miss square only.
3. The **👑 blackout-crown one-away callout** emphasis (D-15) — distinct, higher-intensity than the
   line one-away (which uses 🔥 + accent glow at normal intensity).
4. Deal-screen wordmark/title emphasis (optional, single element).

**NOT accent (data-semantic / celebration language — "data semantics never chrome", the app's B3
principle). These are NOT part of the 10% accent budget:**

| Semantic | Hex | Source precedent | Usage this phase |
|----------|-----|------------------|------------------|
| Caught / positive green | `#22C55E` | CometTrail / SyncDot online / CatchUpSheet | Marked-square stamp fill (✦), healthy fill-meter bar, completed win badges |
| Soft-warning amber | `#F59E0B` | SyncDot schema-drift ("info, not error") | Fill meter amber state when expected fill drops below the likely-line threshold (D-12) — a quiet guide, never a blocker, never red |
| Muted text | `#a1a1aa` (`--color-text-muted`) | app-wide | Fire-rate hints, sublines, "a stretch" bucket label, greyed/deduped swap items (D-04), locked-control explainer (D-10) |
| Rarity tier hues | `config.dex.tierColors` (debut→legendary) | shipped `tierColors` map | Bust-out 🌟 / rare-song emphasis on swap chips **only if** it reads as data (optional); DO NOT recolor board squares by rarity — squares are events/albums/songs, not single-song rarity |

Contrast: near-black on-stamp text uses `#0C0C10` (verified ≥4.5:1 on `#22C55E` per the shipped
`RARITY_ORB_TEXT_COLOR` precedent). Amber `#F59E0B` clears WCAG 1.4.11 non-text contrast (proven
in SyncDot). The win badge / one-away treatments must never rely on color alone — the **word/glyph
always renders** (WCAG 1.4.1): "🔥 One away", "✨ Four corners!", "BLACKOUT!" carry meaning in text.

## Interaction & Motion Contract

This phase is dominated by interaction/motion; the planner + executor consume the specifics below.
All animation is **purely visual — no sound, no haptics** (D-19). All motion reuses the established
`@keyframes`-inside-`@media (prefers-reduced-motion: no-preference)` idiom in `styles.css` and the
already-threaded `useReducedMotion()` hook (D-20).

### Layering (extend `config.ui.z`, no scattered `z-NN`)
| Surface | Tier | Behavior |
|---------|------|----------|
| Peek strip (D-21) | in-flow on ShowView (like the comet trail) — NOT fixed | Sits within the show column; never overlaps the FAB or orbit predictions |
| Mark-toast / badge-toast (D-16/D-18) | `toast` (20) | App-wide, non-blocking, fires over any tab (BackupToast precedent) |
| Supernova overlay (D-17) | new `celebration` tier, **strictly below `sheetScrim` (40)** | `pointer-events-none` — the user keeps logging through it (the live loop is sacred); auto-fades ~2–3s; an open sheet still covers it |
| Full board (peek-strip expand, D-21) | route to `#/games` GizzGames tab (reuse `useHashRoute`) — the simplest, no new overlay tier (Claude's-discretion default) | Tapping the strip navigates to the full board |

### Motion timings (Claude's discretion within D-17/D-18/D-20)
| Element | Motion (reduced-motion fallback) |
|---------|----------------------------------|
| Per-square stamp (D-16/D-18) | ~250ms opacity + slight scale-pop (**reduced: opacity fade only, no scale/translate**) |
| Auto-mark toast (D-16) | fade-in 200ms → hold ~1.2s → fade-out 400ms (~1.8s) (reduced: crossfade, same timing) |
| Badge toast (D-18) | fade/slide-in 200ms → hold ~1.6s → out 300ms (~2s) (reduced: crossfade only) |
| One-away glow (D-13) | slow accent-ring pulse on the single target square (reuse `orb-ripple`/`orb-breathe` idiom) (**reduced: static accent ring, no pulse**) |
| Supernova (D-17) | galaxy-backdrop + orb-burst bloom-in ~400ms → hold ~1.5s → fade-out ~800ms (≤2.7s, inside the 2–3s budget). Reuse `OrbitStage` orb renderer + `ConstellationCanvas` backdrop. (**reduced: static full-bloom badge that crossfades in / holds / out — same information + payoff, zero particles/scale/translate, per D-20**) |

### Celebration hierarchy (D-18 — three tiers, budget-disciplined)
1. **Stamp** — every mark. Cheap per-square opacity/scale.
2. **Medium badge toast** — four-corners, X, and *subsequent* lines after the first. ~2s, non-blocking.
3. **Supernova** — first line + blackout ONLY. ≤2 big moments/show. Non-blocking overlay (D-17).

One-away buildup fires for **lines + blackout only** (D-15); four-corners / X get **no** near-miss
drum-roll but still celebrate on completion.

### Accessibility (D-24)
- Each of the 16 squares is a **labeled, focusable toggle button** (marked / unmarked + lit-by
  label), honoring the Phase-8 labeling discipline. Focus ring uses accent `#f2c14e`.
- **NO ARIA live-region auto-announcements** for marks / one-away / wins in v1 (proportionate for a
  <10-user personal tool; owner is not a screen-reader user). Re-openable later.
- Reduced-motion path (D-20) is a first-class equivalent, not a degraded burst.

# Phase 20 — UI Design Contract

> Visual and interaction contract for **Presence & Interactions** — live ephemeral awareness
> (online dots + coarse tab-level activity), targeted/broadcast waves + a fixed emoji reaction
> palette, and presence-aware Friends rows. Generated by gsd-ui-researcher; verified by
> gsd-ui-checker.
>
> **This phase is "reuse, don't reinvent" (like Phase 19).** Every token below is either DETECTED
> from the shipped Guezzer design system (`packages/app/src/styles.css` `@theme`, `config.ts`) or
> DERIVED from the 21 locked decisions in `20-CONTEXT.md` (D-01..D-21) + `20-RESEARCH.md`. The
> phase introduces **zero new chrome color, no new type role, no new spacing unit, no new z-index
> tier, and no new component library.** New surfaces echo shipped ones — the wave toast mirrors
> `BingoCelebration`, presence fills the **already-reserved** `FriendRow` slots, the sender glyph
> reuses `IdentityGlyph`, the online dot reuses `SyncDot`'s positive-green language. Nothing was
> re-asked of the user; all fields are pre-populated from upstream.

## Spacing Scale

Tailwind v4 default 4px scale — already the app-wide contract (multiples of 4 only).

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs | 4px (`gap-1`, `py-0.5`) | Online-dot → glyph gap, emoji-chip inner padding, toast icon → text gap |
| sm | 8px (`gap-2`, `p-1`, `h-2 w-2`) | The online dot size (8px), palette chip gaps, reaction-palette row padding |
| md | 12–16px (`gap-3`, `px-4`, `py-3`) | Reaction palette container padding, target-picker row padding, toast internal padding |
| lg | 24px (`pb-6`) | Palette section break (emoji row → target-picker row), FriendDetail wave-button spacing |
| xl | 32px | Not required this phase (no full-screen hero introduced) |

Exceptions:
- **44px minimum tap target** (`min-h-11 min-w-11`) on every interactive element — each of the 4 emoji chips, each target-picker row (Everyone + each friend), the "React"/wave entry control on the Friends header, and the FriendDetail pre-targeted wave button. Non-negotiable (SHOW-02 venue "one-thumb-in-the-dark" floor, held app-wide).
- **8px online dot** (`h-2 w-2`, `rounded-full`) — a decoration inside the ≥44px friend/self-row button; the dot itself is not the tap target. Occupies the pre-shipped `data-slot="presence-online"` leading span in `FriendRow`/`SelfRow`.
- **32px identity glyph** (`h-8 w-8`) — reused verbatim from `IdentityGlyph` in the wave toast sender treatment; the toast is `pointer-events-none`, so no hit-area floor applies to the glyph itself.
- **Safe-area insets** honored via `env(safe-area-inset-*)`. The wave-toast host registers its height through `useBottomOverlayHeightRegistration` (BingoCelebration/BackupToast precedent) so it never covers the live-log loop; top-anchored controls use `calc(env(safe-area-inset-top) + 12px)`, never a bare/doubled inset.

## Typography

App-wide roles (unchanged — this phase adds none). Exactly **2 weights**: regular **400** + semibold **600**.

| Role | Size | Weight | Line Height | Usage in this phase |
|------|------|--------|-------------|---------------------|
| Heading | 20px (`text-[20px]`) | 600 | 1.2 (`leading-tight`) | Reaction-palette sheet/popover title ("Send a reaction"), target-picker section heading |
| Body | 16px (`text-base`) | 400 | 1.5 (`leading-normal`) | Wave-toast sender line (`{name} 👋`), target-picker friend-row names, palette helper copy |
| Label | 14px (`text-[14px]`) | 600 | 1.25 (`leading-tight`) | "React"/wave entry affordance label, `🎯 caught it!` chip label, target-picker "Everyone" label |
| Caption | 13px | 400 | 1.25 | The coarse **activity label** in the `presence-activity` slot (`At a show 🎸`, `GizzVerse`, `idle`, `offline`) and the "to you" targeted-wave sub-mark |

Notes:
- The **13px caption** is a **pre-existing app-wide token** (avatar initials, sign-out subline, Phase-19 offline marker) — ratified again here for the activity label / "to you" mark, NOT new scale drift. Do not promote it to a primary role.
- **Emoji render at the surrounding text size** (no oversized emoji) — the reaction chips scale their emoji to ~20px within the 44px chip for legibility; toast/activity emoji inherit their line's size.
- Friend-supplied `display_name` renders **escaped React text, `truncate`/`min-w-0` clamped** (D-11 / T-06-26) — never `dangerouslySetInnerHTML`, never width-unbounded, in both the toast and the target picker. The sender name is resolved from the **trusted friends store by `from` userId**, never taken from the wave payload (V5 input-validation, RESEARCH §Security).

## Color

Dark-only palette from `@theme` (`styles.css`). 60/30/10 split unchanged; this phase adds **no new surface or chrome hue**.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0c0c10` (`--color-surface`) | App background behind the Friends screen + the `pointer-events-none` wave-toast layer |
| Secondary (30%) | `#17171f` (`--color-elevated`) | Reaction-palette surface, target-picker rows, the wave toast card, `At a show` chip background |
| Accent (10%) | `#f2c14e` (`--color-accent`) | **Reserved — see explicit list below** |
| Destructive | `#ef4444` (`--color-destructive`) | **Unused this phase** — no destructive action exists (presence/waves are ephemeral; no delete, no reset, no history to clear). Declared, not applied. |
| Text primary | `#f5f5f7` (`--color-text-primary`) | Sender names, target-picker names, `At a show 🎸` label emphasis |
| Text muted | `#a1a1aa` (`--color-text-muted`) | Coarse activity labels (`LiveGizz`/`GizzVerse`/`idle`), `offline` self-status, secondary sublines |
| Hairline | `#2a2a34` (`--color-hairline`) | Palette/target-picker dividers |

**Accent (`#f2c14e`) reserved for — explicit list (never "all interactive elements"):**
1. **Focus rings** (app-wide a11y, unchanged) — including the emoji chips, target-picker rows, and the wave-entry control.
2. **Selected-emoji indication** in the palette — the chosen chip gets an accent ring/outline during the pick-emoji → pick-target flow (the one selection cue).
3. **Selected-target indication** in the target picker (the chosen "Everyone" or friend row).

No accent fill on the wave toast, no accent online dot, no accent friend-row surface. (If the executor prefers a single accent send-confirm affordance, it counts against this budget and must stay one element.)

**Data-semantic palettes — exempt from the 10% accent budget (B3: "data semantics never chrome"):**

| Palette | Values | Usage — carries meaning, never a button/surface/focus color |
|---------|--------|------------------------------------------------------------|
| Presence / online | **`#22C55E`** (`--color-online`-equivalent, the shipped **SyncDot online green** — owner-ratified 2026-07-14, commit `c99ea42`) | The **binary online dot** in the `presence-online` slot (present-now only, D-14). Same positive-green the app already uses for "connected/online" (SyncDot, CometTrail hit). Dot is 8px, `rounded-full`. **Never** shown when the viewer is offline (D-16) — honest absence, never a stale-green lie. |
| Identity | `#7DD3FC · #FDA4AF · #86EFAC · #FCD34D · #C4B5FD · #5EEAD4` (`config.auth.IDENTITY_COLORS`, via `identityColorIndex(userId, len)`) | The **wave-toast sender glyph** fill (D-09) — reuse `IdentityGlyph` verbatim, stable per user across devices (AUTH-07). Initials always render in `#0C0C10` (identity is never color alone). `config.ts:773` already reserves this palette for "Phase 20 presence dots." |

The online-green and identity palettes are the app's shipped "connection + people" language — do NOT recolor them, do NOT route them through `--color-accent`. The `At a show 🎸` signal is conveyed by **text emphasis (`text-primary`) + the 🎸 emoji**, NOT by adopting an accent or a second green — it stays inside the coarse activity label, honest and flag-only.

Contrast: online green `#22C55E` clears WCAG 1.4.11 non-text contrast on `#0c0c10`/`#17171f` (proven in SyncDot). Sender initials `#0C0C10` on every identity hue verified ≥4.5:1 (shipped `IdentityGlyph`). The online dot **never conveys state by color alone** — it pairs with the visible activity label; when a friend is present the row shows both a dot and a text status (WCAG 1.4.1).

## Copywriting Contract

New strings live in a **`config.copy.presence`** block (single-config-file ethos, CLAUDE.md) plus a `config.presence` constants block (`QUEUE_CAP`, `TOAST_MS`, `DRAIN_GAP_MS` — RESEARCH recommends `{ QUEUE_CAP: 4, TOAST_MS: 1600, DRAIN_GAP_MS: 150 }` as tunable starting values). All sender/emoji strings render as **escaped React text**, never `dangerouslySetInnerHTML` (D-11).

| Element | Copy |
|---------|------|
| Primary interaction (D-04) | **"Send a reaction"** — a single unified surface: pick an emoji, then pick a target. Entry is a compact **"React" / wave affordance** on the Friends screen (header control) + a pre-targeted button inside `FriendDetail` (D-07). Not a full-width accent CTA. |
| Reaction-palette title | `Send a reaction` |
| Fixed emoji palette (D-05/D-06, exactly 4, wave-led) | `👋` (wave, default/first chip) · `🔥` (fire) · `🦎` (lizard) · `🎯 caught it!` (the 🎯 chip carries the "caught it!" label; the other three are emoji-only) |
| Target picker (D-04) | `Everyone` (→ broadcast `to:null`) and one row per online/known friend (→ targeted `to:userId`). Everyone is the top/default option. |
| FriendDetail wave button (D-07) | `Wave at {name}` (pre-targeted at that friend; shares the one `sendWave(emoji, to)` path) |
| Received toast — broadcast (D-09) | `{name} 👋` (or `{name} 🔥` / `{name} 🦎` / `{name} 🎯` per emoji) — reads as group hype |
| Received toast — targeted (D-09) | `{name} waved at you 👋` — with a subtle **"to you"** caption/mark; feels personal (PRES-05). For non-wave emoji: `{name} reacted to you 🔥` (or keep `{name} 🔥` + the "to you" mark — executor's choice, must distinguish targeted from broadcast) |
| Coarse activity labels (D-01/D-03) | `At a show 🎸` (active tracked show on LiveGizz — the residency payoff) · `LiveGizz` · `GizzVerse` · `GizzMap` · `GizzDex` · `GizzGames` · `idle` (tab backgrounded / screen locked, D-02) |
| Self-row status (D-15) | Own live activity label, same vocabulary; e.g. `You` row shows `At a show 🎸` / `GizzDex` etc. |
| Offline self-status (D-17) | `offline` — muted; the "You" row admits you're not broadcasting. Reuses `useOnlineStatus`. |
| Offline / dead-signal (D-16) | **No presence copy shown** — all online dots + activity labels hidden (honest "can't see presence right now"). The Phase-19 `Offline · as of {time}` dimmed PROG rows still render (unchanged — that copy is owned by Phase 19). |
| Empty / no-one-online | **No dedicated empty state.** Presence decorates existing PROG rows only (D-13); when nobody is online the rows simply show no dot + no activity label — never a "nobody's here" panel, never placeholder online-only rows. |
| Error state | **No error surface.** Malformed inbound presence entries / wave payloads are silently skipped at the read boundary (`validateWave` → `null` → dropped, never thrown — RESEARCH §Security V5/V7). A hostile payload never crashes the toast host or the Friends list; there is no user-facing error copy. |
| Destructive confirmation | **None** — this phase has no destructive action (waves are fire-and-forget and ephemeral; nothing persisted to delete or reset, D-21). |

## Interaction & Motion Contract

All motion reuses the established `@keyframes`-inside-`@media (prefers-reduced-motion: no-preference)` idiom in `styles.css` + the already-threaded `useReducedMotion()` hook (D-11). **Visual only — no sound, no haptics** (D-12).

### Layering (reuse `config.ui.z` — no scattered `z-NN`, no new tier)
| Surface | Tier | Behavior |
|---------|------|----------|
| Wave/reaction toast (D-11) | **`toast` (20)** | App-wide, fires over any tab (BackupToast/BingoCelebration precedent), `pointer-events-none`, **never intercepts a tap** — the live logging loop stays sacred (the D-17 Bingo rule). An open `Sheet` (`sheetScrim` 40) still covers it. |
| `ReactionPalette` | `sheet` (50) if rendered as a `<Sheet>`, else an in-flow popover on the Friends screen | Modal palette uses the shipped `Sheet` tier; a lightweight header popover stays in normal flow. Executor's choice — must not invent a new z-tier. |

### Motion timings (Claude's discretion within D-10/D-11)
| Element | Motion (reduced-motion fallback) |
|---------|----------------------------------|
| Wave toast enter/exit | fade + slight rise-in ~200ms → hold `config.presence.TOAST_MS` (~1600ms) → fade-out ~300ms (**reduced: opacity-only crossfade, no translate** — mirrors `BingoCelebration.tsx` reduced path) |
| Burst drain (D-10) | queue drains **one-at-a-time**, `config.presence.DRAIN_GAP_MS` (~150ms) between pops, so a flurry reads as distinct pings (`Matt 🔥`, `Sam 🦎`) rather than a clobber. Over-cap (`> QUEUE_CAP`, ~4) overflow **dropped** (consistent with ephemeral "right now" gestures). |
| Online dot appear/disappear | instant (presence join/leave) — no animation required; if any, a gentle opacity fade, reduced-motion-safe |
| Emoji chip / target selection | accent focus/selected ring, no motion required |

### Behavior states
| State | Behavior |
|-------|----------|
| Online (viewer online, friend present) | Friend row shows the 8px green dot + coarse activity label in the reserved slots. `You` row shows its own dot + live activity (D-15). |
| Live activity change | A friend's dot/label updates live via presence-sync; a calm value swap, never a flash. Multi-tab/device reduces to one activity per user (`atShow` wins, else last entry — RESEARCH Pitfall 2). |
| Receive a wave | Toast pops app-wide via the FIFO queue (D-10/D-11); targeted waves get the "to you" emphasis (D-09); broadcast reads as `{name} 👋`. |
| Send a wave | `ReactionPalette`: pick emoji → pick target → one `sendWave(emoji, to)` call. `to:null` = Everyone, `to:userId` = one friend (D-04/D-05). No send rate-limit (D-08); the receive-side cap bounds on-screen flooding. Sender does **not** self-toast (`broadcast.self` off + defensive `from===me` drop — RESEARCH Pitfall 3). |
| Viewer offline (dead-signal venue, D-16) | **Hide ALL online dots + activity labels** — never a stale-green lie. Phase-19 dimmed `Offline · as of {time}` cached PROG rows still render (name + stats), so the screen is never blank. |
| Self offline (D-17) | The `You` row reads `offline`, own dot hidden — honest that you're not broadcasting. Reuses `useOnlineStatus`. |
| Malformed inbound payload | Skipped at the read boundary (`validateWave`/presence-entry validation), never crashes the host (V5/V7). |
| Missed wave (recipient offline/app closed, D-21) | Simply gone — no queue, no badge, no replay, no history. A wave is a "right now" gesture. |

## Accessibility

- Every interactive element ≥44px hit area (`min-h-11 min-w-11`) — emoji chips, target rows, wave entry, FriendDetail wave button.
- **Online status never by color alone** — the green dot always pairs with the visible activity label (WCAG 1.4.1); an offline viewer sees neither (honest absence).
- Identity conveyed by **initials + color** in the toast glyph (never color alone).
- Wave toast is `role="status"` (calm, non-interrupting), `pointer-events-none`, reduced-motion-aware (opacity-only fallback), never blocks the live loop.
- Sender/emoji/name strings are **escaped React text**, `truncate` clamped; icon-only controls (wave entry) carry `aria-label`.
- Emoji chips are labeled toggle/press targets with accessible names (`aria-label="Wave"`, `"Fire"`, `"Lizard"`, `"Caught it"`); the 🎯 chip's visible "caught it!" label doubles as its name.
- Proportionate to a <10-user personal tool: **no ARIA live-region auto-announcements** for incoming waves (matches the Phase-16 D-24 decision); the visual toast is the channel. Re-openable later.

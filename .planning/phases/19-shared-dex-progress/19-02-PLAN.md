<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → Supabase `progress` table | Every write crosses the RLS boundary; a client can only insert/update its own row (`auth.uid() = user_id`) |
| synced friend rows → app read boundary | Untrusted `summary` jsonb + `display_name` from any friend row enters via `refreshAllFriends` before caching/rendering |
| offline cache → app | Last-known friend data persisted in Dexie, read when offline |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-authz | Tampering / Elevation (V4 Access Control) | `upsertOwnProgress` → `public.progress` | mitigate | Server-side RLS `with check (auth.uid() = user_id)` (provisioned Phase 17) makes cross-user inflation structurally impossible; the client keys the upsert by `useAuthIdentity().userId` only. Verified manually in 19-04, not re-created here (PROG-03). |
| T-19-identity | Tampering | identity-only upsert path | mitigate | Identity-only writes touch `{user_id, display_name}` only, never `summary`; content writes always send a full fresh `summary`. Pinned by the `upsert` test (no-`summary` assertion) (D-15, Pitfall 4). |
| T-19-tamper | Tampering / DoS (V5 Input Validation) | `refreshAllFriends` re-pull | mitigate | Each row's `summary` runs through core `parseSharedProgress`; malformed/hostile rows are skipped, never crash the pull; array/numeric clamps live in the zod schema (D-19). Pinned by the `subscribe` skip-malformed test. |
| T-19-realtime | Denial of Service (Pitfall 2) | `postgres_changes` subscription | mitigate | Publication already added Phase 17 (do not remove); subscription is re-established on the reconnect signal so a dropped socket does not silently stop live updates (D-17). |
| T-19-SC | Supply chain (Tampering) | npm installs | accept | No new packages installed (`@supabase/supabase-js`, `dexie`, `zod` all vetted/installed in Phases 17/18/1). No install surface; slopcheck N/A. |
</threat_model>

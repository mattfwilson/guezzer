## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| imported file → SettingsView → core → Dexie | A user/friend file crosses into the local dex |
| location.hash → route selection | Hash input selects a view (allow-list controlled) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-15 | Tampering | importPicker → parseAndMergeImport | mitigate | Validate-then-merge in core; on ok:false NO DB write; success commits atomically via db.importSnapshot (D-12, Pitfall 5) |
| T-05-16 | Tampering / XSS | SettingsView / ImportResult rendering | mitigate | Result/error strings and any imported values rendered as escaped React text only; no dangerouslySetInnerHTML |
| T-05-17 | Tampering | route selection (ROUTES allow-list) | mitigate | "settings" added to the validated fixed allow-list; hash only ever selects a view, never assigned to innerHTML/location/eval (T-03-02 preserved) |
| T-05-18 | Denial of Service (data loss) | export as eviction backstop | mitigate | Prominent always-available Export + End-Show auto-download + one-time persist-denied warning (PWA-04/D-13) |
| T-05-19 | Repudiation | destructive controls | mitigate | No clear/reset control shipped (D-14) — no accidental data-loss surface on a hard-bar phase |

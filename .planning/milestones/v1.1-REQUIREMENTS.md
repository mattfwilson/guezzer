# Requirements Archive: Guezzer — v1.1 Polish & Pre-Show Hardening

**Core Value:** At a live show, with one thumb, in the dark, the user can see credible next-song predictions and log the setlist as it happens — fully offline once loaded.

**Milestone goal:** Close the v1.0 audit's non-blocking gaps and prove the app show-ready on real hardware before show #1 (late Aug/Sep 2026). No new user-facing features — a low-risk hardening pass.

## v1.1 Requirements — Outcomes

### UI Polish

- [x] **POLISH-01** — Prediction-orb and center-node song names render fully legible on a small phone screen. **Outcome: VALIDATED (Phase 8).** `fitOrbLabel` made circle-aware (per-line chord width + stacked-height budget incl. the percent line), guarded by a geometric sweep over the real [56..112]px diameter range; confirmed on-device via `#/dev/orb-fit`. A small-orb overflow regression found on-device was closed by the 08-08 gap-closure plan.
- [x] **POLISH-02** — Show-Mode FAB speed-dial (D-20) and once-per-version InstallBanner (D-22) verified against their originating todos; todos moved to resolved. **Outcome: VALIDATED (Phase 8).** Both confirmed matching intent (landed Phase 6); originating todos formally resolved.

### Accessibility

- [x] **A11Y-01** — Every bottom sheet and modal dialog dismisses with Escape and manages focus (trap while open, restore to trigger on close). **Outcome: VALIDATED (Phase 8).** All 7 surfaces (NodeSheet, AppMenu, TrailNodeSheet, EndShowDialog, ShareCardSheet, "Whose dex is this?" prompt, CompareView) migrated onto a shared `<Sheet>` primitive; confirmed with VoiceOver + external keyboard on iOS, incl. `inert` propagation through `display:contents`.
- [x] **A11Y-02** — While a constellation node is focused, the NodeSheet does not occlude the Explore FilterFab. **Outcome: VALIDATED (Phase 8).** FilterFab lifted above the sheet peek via `config.ui.z` + `FAB_SHEET_GAP_PX`.
- [x] **A11Y-03** — Resizing the viewport with a node focused keeps the camera framed. **Outcome: VALIDATED (Phase 8).** Shared visible-viewport source + resize-reframe; no snap-off through rotate/keyboard resize.

### Data Integrity

- [x] **DATA-06** — `shownotes` carried verbatim through ingestion AND normalization per SCHEMA.md §12, with an automated survival test. **Outcome: VALIDATED (Phase 9).** Position-1-wins denormalization, within-show-disagreement stats counter, byte-for-byte carry; `corpus.json` regenerated for all 738 shows with byte-stable downstream artifacts (archive.json byte-identical inside its 250 KB budget). Resolves audit WR-01.

### Data Safety

- [x] **PWA-05** — On a fresh/evicted DB, typing your own owner name into the "Whose dex is this?" prompt restores (merges) your backup. **Outcome: VALIDATED (Phase 9).** Pure `isTypedNameMine` helper routes the name-match path to merge; real-Dexie union-merge proof shows local data is never dropped. Polishes the WARNING-1 fix from quick 260716-vw2.

### Pre-Show Validation

- [x] **VALID-01** — Owner completes the tuning-family tag spot-check (DATA-04). **Outcome: VALIDATED (Phase 10).** Read-only tuning-review CLI drove the review; owner confirmed 12 canonical anchors + 36 no-album-default hand-tags; D-03 fix re-tagged 9 *Infest the Rats' Nest* tracks standard→cs-standard with zero backtest top-k regression.
- [x] **VALID-02** — Full show-loop dry-run passes on a real device before show #1, with Android exercised if available. **Outcome: VALIDATED (Phase 10).** The complete loop (start → predict → log hits/misses → set break → encore → End Show → recap → dex credit) plus the offline airplane-mode leg and JSON export/import round-trip all passed on-device on iPhone over a cloudflared HTTPS tunnel (2026-07-19). Two D-09 blockers found+fixed inline; a per-show recap Share card added. Android waived — no device (D-06).

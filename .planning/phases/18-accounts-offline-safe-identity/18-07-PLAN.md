<objective>
Close BOTH halves of AUTH-05's local-data isolation (D-08/D-09/D-11). Plan 02 added the `userId` field/index + the one-time claim and Plan 06 scopes `useDexStats`, but three gaps remain: (1) four other namespaced-table consumers still read UNSCOPED — a borrowed-phone signer (identity B) would see identity A's shows, entries, and bingo; (2) export/import still read/write ALL rows unscoped; and (3) — the load-bearing fix — **no create path stamps `userId`**, so every helper (`startShow`, `logSong`, `adoptSuggestion`, `markShowAttended`, `saveDraftCard`) writes rows with `userId` undefined, which the scoped reads and scoped export then EXCLUDE — meaning the signed-in user's own post-claim activity would silently vanish from their Dex/Shows/Recap/Games and backups (regressing v1's core loop). This plan (a) scopes every namespaced-table read in `ShowsList`, `ArchiveBrowser`, `RecapView`, and `GamesView`; (b) stamps the signed-in `readIdentityRecord()?.userId` on all five create paths via Dexie `creating`/`updating` hooks in `db.ts` (no helper-signature change — the lowest-touch mechanism); and (c) changes `snapshot()`/`importSnapshot()` to require a `userId` and wires `readIdentityRecord()?.userId` into their two non-React callers — all in one plan so `tsc` never sees a signature/caller mismatch. It runs in Wave 2 (needs Plan 02's `userId` field + Plan 03's identity accessors), completing BEFORE Plan 06's Wave-3 device-UAT so the on-device cross-identity isolation check (a second friend sees an empty dex) is actually true across the whole app.

Purpose: The offline-safe identity is worthless if any single unscoped read leaks another friend's dex on a shared phone — OR if the signed-in user's own new catches vanish because they were never stamped with an identity. This plan owns the remaining read leak surface AND the write-side stamping so D-09 holds everywhere and the core loop does not regress for the signed-in user.
Output: scoped `ShowsList.tsx`/`ArchiveBrowser.tsx`/`RecapView.tsx`/`GamesView.tsx`; userId-stamping create hooks + scoped `snapshot`/`importSnapshot` in `db.ts`; wired `exportDownload.ts`/`importPicker.ts`; `authViewScoping.test.tsx`, `authWriteStamping.test.ts`, and an export→import round-trip isolation assertion.
</objective>

<objective>
Add the bingo barrel exports and build the calibration gate CLI — the Bingo equivalent of the Phase-2
backtest trust gate (D-19). `bingo-calibrate.ts` reads the shipped corpus/matrix/archive/rarity/dex-albums
artifacts, builds one `BingoContext`, deals N seeded cards per vibe, replays all 241 recent-era shows
(year≥2022) through the REAL `deriveMarks` + `detectWins` (no forked logic — Pitfall 3), aggregates
P(line)/P(blackout)/dark-square-share/expected-marks per vibe, prints the human-trust report (D-19a),
and hard-asserts the D-02/D-03/D-05 invariants — `process.exit(1)` on any violation (D-19b). It also has
a `--candidates` mode that emits jam-vehicle + album roster candidates to a review file (never config.ts)
for the D-20 checkpoint. This plan builds the machinery; Plan 06 RUNS the D-20 process and locks constants.

Purpose: The gate makes un-calibrated config unable to silently ship. Building the sim on the real fold
guarantees the reported numbers match production cards.
Output: index.ts barrel block, cli/bingo-calibrate.ts, calibrate.test.ts.
</objective>

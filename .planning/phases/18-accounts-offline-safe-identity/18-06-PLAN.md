<objective>
Wire THE CRUX (AUTH-02 / D-05/D-06): interpose the `AuthGate` between `main.tsx` and `<App/>` so the app cold-boots fully offline to the complete dex whenever an app-owned identity record is present — with zero await, no splash, no `getSession()` on the boot path, and gating on identity PRESENCE not token validity. Run `getSession()`/`onAuthStateChange` only in the background to reconcile connectivity and drive the calm reconnecting affordance. Complete the AUTH-05 read half by scoping `useDexStats` to the current user id, and complete AUTH-04 by making sign-out an instant teardown. This is the highest-risk seam and the non-negotiable trust gate for Phases 19–20 — it ends with an on-device offline cold-boot verification.

Purpose: A friend whose token expired overnight must never be locked out of their own offline dex at a dead-signal venue. Startup must not regress the shipped v1 synchronous offline boot.
Output: `AuthGate.tsx`, integrated `main.tsx`/`App.tsx`, scoped `useDexStats.ts`, reconnecting wiring in `ShowView.tsx`, `authGate.test.tsx`, `authNamespacing.test.tsx`, and a passed device-UAT of the crux.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| boot decision ← app-owned identity record | Boot trusts the local identity record's presence to open the offline dex |
| background reconciler ← Supabase auth events | Network auth events reconcile connectivity but must never gate boot |
| identity A ↔ identity B on one device | Scoped reads must isolate the current identity's dex |

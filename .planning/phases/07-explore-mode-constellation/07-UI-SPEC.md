# Phase 7 — UI Design Contract

> Visual and interaction contract for Explore Mode: the force-directed transition constellation (`#/explore`) rendered from the same matrix artifact the predictor uses, with rotation/threshold filters, tap-a-node ranked next-song bars, focus+context highlighting, settle-and-freeze physics, and the personal dex overlaid. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **This phase INHERITS the Phase 3 + 4 + 5 + 6 foundation** (`03-UI-SPEC.md`, `04-UI-SPEC.md`, `05-UI-SPEC.md`, `06-UI-SPEC.md`): spacing scale, typography (2 weights / 4 sizes), the dark-only 60/30/10 chrome palette, the tuning-family data palette, the hit/miss semantic pair, the rarity ramp, the 44px tap floor, `lucide-react` icons, the system font, the FAB idiom (06 D-20), the bottom-sheet idiom (TrailNodeSheet / ShareCardSheet), and the dimmed-silhouette language (06 §B4, declared there as "the same visual language Phase 7's constellation dimmed-silhouettes will inherit"). **Tokens are NOT re-derived here — they are reused verbatim.** New for Phase 7: the canvas rendering surface (first non-DOM view), canvas node/edge/label/ring styling rules, the focus-dim vs dex-dim opacity contract, and the data-driven defaults for rotation window N and the edge-count threshold (closes CONTEXT D-06/D-07 and PROJECT.md open question #2).

## Spacing Scale

Inherited verbatim from Phase 3 (multiples of 4, Tailwind v4 4px base). **Do not re-declare in code — reuse the existing `@theme` scale.**

| Token | Value | Usage (Phase 7 additions) |
|-------|-------|---------------------------|
| xs | 4px | Bar label-to-% gap, ring-to-node offset, filter-panel row internal gaps |
| sm | 8px | Bar row vertical gap, filter-panel control spacing, sheet grab-handle margin |
| md | 16px | Sheet gutters, filter-panel padding, bars-panel header padding |
| lg | 24px | Sheet section rhythm (header → bars list) |
| xl | 32px | Error/empty-state vertical rhythm |
| 2xl | 48px | — (reserved, unused this phase) |
| 3xl | 64px | Error-state top spacing |

**Exceptions (functional, inherited + extended):**

- **44px minimum interactive target** (inherited hard floor). DOM surfaces: filter FAB (56px, `ui.FAB_DIAMETER`), every filter-panel control row (toggle halves, overlay switch, slider thumb hit area ≥ 44px tall), sheet grab handle, every ranked-bar row (bars are tappable chain-hops per D-16 — full-row hit area ≥ 44px), "Show all N" expander.
- **Canvas node hit area ≥ 44px equivalent:** canvas nodes are hit-tested by painted pointer area, not DOM boxes. The `nodePointerAreaPaint` circle MUST be at least **22px screen-space radius** (44px diameter) at every zoom level, regardless of the node's visual radius — `explore.NODE_HIT_MIN_RADIUS_PX = 22`, enforced by dividing by `globalScale` in the pointer-area draw. Small nodes look small but never tap small (same principle as SHOW-02's orb floor).
- **Filter FAB anchor:** `position: fixed`, bottom-right; bottom offset = `env(safe-area-inset-bottom)` + BottomTabBar height + `sm` gap; right offset = `md` + `env(safe-area-inset-right)`. Same anchor contract as the Show-Mode FabMenu **minus the SuggestionStrip slot** (the strip is Show-Mode-only chrome and never renders at `#/explore`).
- **Bottom sheet peek height = 40% of viewport** (`explore.SHEET_PEEK_FRACTION = 0.4`, D-14) so the focused node and its lit neighborhood stay visible above the sheet; full state = viewport minus header + safe-area. Fractional heights are layout geometry, not spacing tokens.
- **Safe-area insets** via `env(safe-area-inset-*)` — inherited; the canvas fills edge-to-edge but the FAB, sheet, and filter panel respect insets.

## Typography

Inherited: **two weights only** (regular 400, semibold 600), **four sizes**. No new sizes or weights. Phase 7 usage mapping:

| Role | Size | Weight | Line Height | Phase 7 usage |
|------|------|--------|-------------|---------------|
| Body | 16px | 400 | 1.5 | Bar "why" lines, error/empty-state bodies, filter-panel helper text |
| Label | 14px | 600 | 1.4 | Bar song names + percentages (`tabular-nums`), filter-panel control labels, slider value readout, "Show all N" expander, sheet sublines |
| Heading | 20px | 600 | 1.2 | Sheet header (focused song name), error-state heading |
| Display | 28px | 600 | 1.2 | Reserved — not used this phase (the constellation itself is the hero; no number moment outranks it) |

**Canvas label exception (documented functional exception, extends the D-21 orb-label precedent):** node labels are drawn in the canvas 2D context, not DOM text. Rules:

- Labels render at a **constant screen-space size** (`fontSizePx / globalScale` in world units): **12px** regular 400 for zoom-revealed labels, **14px semibold 600** for the focused node and its neighbors (Label-adjacent; canvas data-viz text mirrors the 11px orb-label floor precedent, not the 16px body floor which governs free-flowing DOM text).
- Sighting-count numbers inside rings: **11px** semibold 600, `tabular-nums`-equivalent (canvas digits don't reflow; use consistent metrics).
- Labels never render below 11px screen-space — below that threshold they are simply not drawn (zoom gating, §Layout).
- Label text is ellipsized past **18 characters** (`explore.LABEL_MAX_CHARS`) except on the focused node, which always renders its full name (the sheet header also carries it).

## Color

Dark theme is **mandatory** (inherited — no light theme, no toggle). **Phase 7 introduces ZERO new hues.** Every color below is an inherited token; what is new is the canvas *opacity contract* (dex-dim vs focus-dim) and the accent reserved-list addition for the focused node.

### A. Chrome palette (60/30/10 — inherited, unchanged)

| Role | Value | Usage (Phase 7) |
|------|-------|-----------------|
| Dominant (60%) | `#0C0C10` | Canvas background (the night sky), error/empty-state background |
| Secondary (30%) | `#17171F` | Bottom sheet, filter FAB + panel, bar-row surface |
| Accent (10%) | `#F2C14E` (warm gold) | See reserved list |
| Text primary | `#F5F5F7` | Sheet header, bar names/%, filter labels, focused-node label |
| Text muted | `#A1A1AA` | "Why" lines, helper text, zoom-revealed labels at rest, unseen-node labels |
| Border / hairline | `#2A2A34` | Sheet grab handle, panel borders, bar tracks, FAB border, edge base color |

**Accent gold reserved for (explicit — never "all interactive elements"):**

1. The **focused node's selection ring** on canvas — one node at a time, the constellation's equivalent of the inherited one-active-tab rule. 2px screen-space stroke around the focused node only.
2. The **focus-visible ring** on keyboard/AT focus (inherited).
3. The **active half of the Rotation | Full catalog toggle** in the filter panel (same segment-control accent use as Phase 6's Albums | Shows, reserved-use precedent).

Everything else — the filter FAB (deliberately NOT accent, Phase 6 precedent), slider, overlay switch, bars, expander — uses text-primary/text-muted on the secondary surface. Nodes and edges are colored by the **data palettes below**, never accent. Legendary-gold (Phase 6 B3) does **not** appear in Phase 7 surfaces — the bars panel carries no rarity badges (counts + dates are the honest payload per D-02); one row never carries two competing color systems.

### B1. Tuning-family DATA palette (inherited — node fills)

Reuse `packages/app/src/show/tuningColor.ts` **verbatim** — same module, same `TuningFamily`-union keys (note the code key is `"cs-standard"`, not the display label):

| `tuningFamily` key | Value | Phase 7 usage |
|--------------------|-------|---------------|
| `standard` | `#5EC8E5` (cyan) | Node fill |
| `cs-standard` | `#B98CF2` (violet) | Node fill |
| `microtonal` | `#FF8A5B` (warm coral) | Node fill |
| null / unmapped | `#A1A1AA` (text-muted) | Neutral fallback fill (inherited rule — never invent a color) |

Also used as the **bar fill** in the ranked-bars panel: each bar fills in its *target* song's tuning color at **60% opacity** on a `#2A2A34` track (the bar represents a song; same key, same module). All three families clear ≥ 3:1 graphical contrast on `#0C0C10` (ratified Phase 4).

### B2. Hit-green caught semantics (inherited extension — sighting rings)

`#22C55E` is the app-wide "caught / positive" signal (Phase 6 §B2 sanctioned extension of the owner's `260713-w1c` precedent). Phase 7 usage:

- **Sighting ring (DEX-05, D-11):** every caught node wears a `#22C55E` ring (1.5px screen-space stroke, `xs`-offset outside the node fill) whenever the dex overlay is ON. The **count number** (11px, text-primary) draws centered in a small green-tinted pill above the node **only past the count zoom threshold** (§Layout) — no 264-badge soup at rest.
- **Caught tick on bar rows:** when the overlay is ON, caught target songs in the bars panel show the small green `Check` leading the name (same language as dex SongRow); unseen targets show the hollow `#2A2A34` circle. Overlay OFF hides both — the panel mirrors the sky's current semantics.
- Miss-red `#EF4444` is **unused** this phase (no destructive actions, no miss semantics in Explore). Do not introduce it.

### B3. Edge color (neutral — never tuning-colored)

Edges connect different families; coloring them would lie. Contract:

- Base edge stroke: `#A1A1AA` at **20% opacity**; width scales with transition count: `0.5px + sqrt(count) * 0.5px`, clamped to **4px max** screen-space.
- **Directed:** every edge draws a small directional arrow (`linkDirectionalArrowLength` ≈ 3.5 world units, arrow at 90% along the edge) — direction is the data (EXPL-01), not decoration.
- Neighborhood-highlighted edges (focus state): opacity raised to **70%**, same hue. No new color.

### B4. Dimming contract — two dims, never confused

Two independent dimming systems coexist; their values are deliberately far apart so state reads unambiguously:

| Dim | Trigger | Treatment | Config |
|-----|---------|-----------|--------|
| **Dex-dim (silhouette, D-10/DEX-05)** | Overlay ON + song unseen | Node fill desaturated to its grayscale luminance equivalent at **35% opacity**; no ring; label (when zoom-revealed) at text-muted. Phase 6 §B4 language: dimming signals state, never hides — the node stays tappable and its label stays legible when drawn | `explore.DEX_DIM_OPACITY = 0.35` |
| **Focus-dim (context, EXPL-05/D-13)** | A node is focused | Everything outside the focused node + its direct neighborhood (in/out neighbors + connecting edges) drops to **12% opacity** — nodes, edges, rings, labels alike. The neighborhood renders at its normal (dex-aware) treatment; the focused node adds the gold ring and forced labels | `explore.FOCUS_DIM_OPACITY = 0.12` |

Focus-dim stacks on top of dex-dim (an unseen non-neighbor goes to 12%, not 35%×12%— take the minimum, don't multiply). Toggling the overlay OFF restores every node to full tuning color ("dim = data, not dex", D-10).

## Data-Driven Defaults (D-06 / D-07 — closes PROJECT.md open question #2)

Measured against the shipped artifacts (2026-07-16: `archive.json` 738 shows through 2025-12-13; `transition-matrix.json` 264 nodes / 2,987 edges):

**Rotation window N (`ROTATION_WINDOW_SHOWS = 5`).** Distinct songs played in the last N shows: N=4 → 40, **N=5 → 56**, N=6 → 71, N=7 → 84, N=8 → 86. The D-06 target is a readable default graph of ~40–80 nodes; **N=5 lands mid-band at 56 nodes** and reads naturally ("the last five shows" ≈ the current leg of a tour). N=7 already breaches the band. Config constant (core), UI-invisible per D-12.

**Edge-count threshold default (`EDGE_COUNT_THRESHOLD_DEFAULT = 2`).** Edge count distribution: 1,946 of 2,987 edges (65%) are **one-play edges** — exactly the misleading 100%-from-one-play class D-07 exists to kill. Threshold ≥2 removes all of them by construction, leaving 1,041 edges catalog-wide and **174 edges within the N=5 rotation subgraph (56 nodes / 174 edges — a legible default sky; ≥3 would leave 103)**. Slider range **1–10** (count ≥ X; max observed count is 224 but ≥10 already reduces to 133 edges — deeper cuts add nothing legible). Default 2, minimum 1 restores the full truth for whoever wants it.

Both constants live in core config beside the derivation function; re-run the measurement if the corpus is refreshed.

## Layout & Interaction Contract

### Region anatomy (`#/explore` — replaces `PlaceholderView`)

1. **AppShell chrome (inherited):** header (wordmark + menu) and BottomTabBar unchanged; `#/explore` mounts with **`scroll={false}`** (the Phase-4 non-scrolling full-height seam, same as `#/show`) — the canvas owns all gestures.
2. **Constellation canvas:** fills the remaining viewport. `react-force-graph-2d` in a **single component**; graph data = pure core derivation from the matrix JSON (one pipeline, CLAUDE.md). Node visual radius scales by play count: `NODE_RADIUS_MIN 4` + `sqrt(playCount)` scaling clamped to `NODE_RADIUS_MAX 14` (world units). Fill = tuning color (§B1), ring = sighting ring when caught + overlay ON (§B2), edges per §B3, dims per §B4.
3. **Filter FAB (D-09):** fixed bottom-right per §Spacing anchor. Collapsed: 56px circle, elevated + hairline, `SlidersHorizontal` glyph. Expanded: a compact panel (secondary surface, `md` padding, hairline border) opening upward containing exactly three controls top→bottom: Rotation|Full toggle, edge slider, dex-overlay switch. **No scrim** — the graph must stay visible and live while sliding (D-09: "the graph stays visible while sliding"); tapping the FAB (glyph rotated 45°) or the canvas collapses the panel. Slider changes apply immediately (edge visibility is a render-pass filter — no simulation reheat).
4. **Bars bottom sheet (D-14):** partial sheet at 40% viewport height on node focus; grab handle (hairline, `sm` margin); drag up → full height; swipe down / tap empty canvas → dismiss. Same idiom as TrailNodeSheet/ShareCardSheet. Content: sheet header → top-10 ranked bars (D-04) → "Show all N" expander → the muted D-03 note.

### Interaction rules

- **Tap a node (D-13):** in ONE gesture — focus ring (gold), neighborhood highlight + focus-dim the rest (§B4), forced labels on focused node + neighbors, sheet opens at peek. Camera pans/zooms the focused node into the upper 60% of the viewport (`centerAt` + `zoom`, 400ms ease) so the sheet never covers it.
- **Tap empty canvas:** clears focus + dims, dismisses the sheet, restores the at-rest label state (D-13).
- **Tap a bar (D-16 — chain-hop):** selects that song as the new focus: pans/zooms to it, swaps highlight, reloads the sheet with *its* outgoing bars. Sheet stays at its current snap. This is the core exploration loop — walking probable setlist paths bar-by-bar.
- **The panel always tells the whole truth (D-03):** ranked bars are computed from the node's **complete outgoing edge list** in the matrix — independent of the slider and rotation toggle. A chain-hop target may be a node that isn't currently drawn (full-catalog song while in Rotation view): the focused node is **always rendered regardless of active filters** (temporary filter exemption while focused), so a chain-hop never lands on empty space.
- **Slider hides all of a node's edges (D-08):** the node stays visible as a free-floating star — population never churns with the slider; it remains tappable for its history/dex state.
- **Zoom-gated labels (D-15, answers the STATE-flagged ~250-node spike):** at rest, only the **top K=8 nodes by play count** carry labels (`LABEL_AT_REST_TOP_K = 8`; the planned canvas-label spike may set it to 0 if even 8 jitter-crowds small screens). All labels fade in past `LABEL_ZOOM_THRESHOLD` (`globalScale ≥ 1.5`); sighting-count numbers past `COUNT_ZOOM_THRESHOLD` (`globalScale ≥ 2.5`). The focused node + neighbors are **always labeled regardless of zoom**. Label draws use the §Typography canvas rules.
- **Settle and freeze (EXPL-06):** simulation runs with `d3AlphaDecay 0.035`, `d3VelocityDecay 0.45`, `cooldownTicks 200`; on `onEngineStop`, **fix every node** (`fx = x, fy = y`) — labels never jitter permanently. Pan/zoom never reheats.
- **Filter/overlay changes preserve positions (discretion, decided):** toggling rotation/full or the overlay **never re-runs layout for surviving nodes** (fixed positions kept — the sky must feel stable, per CONTEXT bias). Nodes entering the view (Rotation → Full) spawn near the frozen graph's centroid with small jitter and settle via a **low-alpha reheat that moves only unfixed nodes**, then freeze on `onEngineStop`. Overlay and slider changes are pure render-pass changes — zero physics.
- **Gesture suppression (inherited SHOW-13 scope):** the canvas region carries `touch-action: none` (the library owns pan/zoom/tap), `overscroll-behavior: none`, `user-select: none`; no pull-to-refresh, no rubber-banding. The sheet interior scrolls its bars list normally.
- **Motion:** camera pan/zoom via the library's tweens; sheet + FAB panel are CSS transitions. `prefers-reduced-motion`: camera moves become instant jumps, sheet/panel become opacity swaps, and the settle animation is allowed (it is layout computation, not decoration) but the initial view may render pre-warmed (`warmupTicks`) to shorten visible motion.
- **Live reactivity:** the overlay reads `useDexStats` — marking a show in the Dex tab recolors the constellation on next visit (or live if mounted) with no second derivation path.
- **Accessibility floor:** all DOM text ≥ 4.5:1; canvas graphical elements ≥ 3:1 (tuning fills ratified Phase 4; rings/edges per §B). The canvas gets `role="img"` + `aria-label` "Song transition constellation — {n} songs shown"; all *controls* (FAB, panel, sheet, bars) are DOM and fully keyboard/AT operable; the bars panel is the accessible readout of any focused node's data. Focus-visible ring (gold) on every DOM interactive element.
- **Untrusted strings (inherited):** song names from the artifact render as escaped text / canvas `fillText` only — never HTML injection.
